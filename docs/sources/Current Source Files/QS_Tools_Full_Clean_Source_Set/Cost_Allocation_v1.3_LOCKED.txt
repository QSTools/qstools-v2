# Cost Allocation — Source of Truth (v1.3 — LOCKED)

---

## Purpose

The Cost Allocation Module defines the structural relationship between Labour and Assets so the Cost Summary Module can determine how the business is expected to recover cost.

It is a structural linking layer only.

It does NOT:

* calculate labour cost
* calculate asset cost
* calculate overheads
* store financial outputs
* duplicate Cost Summary logic

It defines:

* how labour and assets are structurally connected
* whether recovery is expected from labour only or from asset-linked delivery
* which active links are valid for downstream commercial logic

---

## ⚠️ IDENTITY RULES

Cost Allocation does NOT own:

* staff_id
* asset_id
* asset_package_id

Cost Allocation may only consume:

* staff_id from Labour
* asset_id from Assets in V1
* asset_package_id from Asset Packages in the future

Rules:

* Labour remains identity owner for staff_id
* Assets remains identity owner for asset-side identity in V1
* Asset Packages may become the package-side identity owner later
* Cost Allocation must never create replacement IDs for upstream records
* display names are display only and must never be used as keys

## 📊 Structural Rules

### asset_driven

Meaning:

* revenue is primarily generated through asset-linked delivery
* labour is structurally attached to an asset in V1
* labour may later be attached to an asset package once Asset Packages exists
* Cost Summary decides recovery logic using these active links

Rules:

* active link records are allowed
* duplicate active links with the exact same asset_id + staff_id are not allowed in V1
* duplicate active links with the exact same asset_package_id + staff_id are not allowed after migration

---

### labour_only

Meaning:

* revenue is primarily generated through labour
* asset linkage is not required for recovery logic
* Cost Summary uses labour pathway without requiring asset-linked structure

Rules:

* asset links may be empty
* existing link history must be preserved
* Cost Summary consumes revenue_model even if no active links exist

---

## 📊 Outputs

The Cost Allocation Module produces structural outputs only:

## 🔗 Integration Rules

### Labour Module

* provides staff_id
* remains the only owner of staff identity

### Employee Overheads Module

* no direct ownership relationship
* not used for linking
* consumed later by Cost Summary only

### Assets Module

* provides asset_id in V1
* remains the owner of asset-side identity in V1

### Asset Packages (Future)

* may replace asset_id as the structural asset-side unit after formal implementation

### Cost Summary Module (CRITICAL)

* consumes active_revenue_model
* consumes active_asset_labour_links[]
* determines recovery logic
* determines commercial outputs

### Budget Module

* consumes Cost Summary outputs, not Cost Allocation directly unless later required for scenario display

### Quote Engine (Future)

* may consume asset_labour_link_id where operational assignment is needed
* must not repurpose cost allocation records into pricing logic
