QS Tools — New Page Build Checklist (v1.0 — LOCKED)

STATUS: DAILY BUILD CHECKLIST

HOW TO USE THIS
Before building → skim Sections 1–3
During build → follow Sections 4–8
Before push → complete Section 9

1. MODULE DEFINITION (BEFORE BUILD)
Confirm:
- Module brief exists
- Module purpose is clear
- Inputs are defined
- Outputs are defined
- Ownership is clear

2. MODULE TYPE (CHOOSE ONE)
Choose:
- Input Module
- Summary Module
- Structural Module

Do NOT invent a new structure.

3. FILE STRUCTURE CREATED
Confirm:
- app/<module>/page.jsx
- hooks/use<Module>.js
- lib/calculations/<module>Calculations.js
- lib/selectors/<module>Selectors.js
- lib/storage/<module>Storage.js
- lib/storage/<module>ProfileStorage.js
- components/<module>/

4. NAMING CHECK
Confirm:
- Hook = use<Module>
- Calculations = <module>Calculations.js
- Selectors = <module>Selectors.js
- Storage = <module>Storage.js
- Components use <Module> prefix
- Variables use snake_case
- IDs use _id format

5. PAGE ARCHITECTURE
Confirm:
page.jsx
- only imports + layout
- calls hook only
- passes props to components
- NO calculations
- NO data shaping

hook
- orchestrates logic
- uses selectors
- prepares status + card

calculations
- all formulas live here
- no UI logic

selectors
- shape data for UI
- build warnings/status

components
- UI only
- no formulas
- no hidden logic

6. DOWNSTREAM CONTRACT CHECK
Confirm:
- using upstream outputs only
- NOT rebuilding upstream maths
- NOT using raw form state downstream
- NOT renaming contract variables

7. UI SYSTEM CHECK
Confirm:
- using UI classes only
- using CSS tokens
- no inline styles
- no text-white, bg-* hacks
- consistent spacing + layout

8. RESPONSIVE / LAYOUT CHECK
Confirm:
- mobile-first layout works
- no random md:grid-cols-*
- collapse used where needed
- follows Phase 3 rules

9. PRE-PUSH FINAL CHECK
Before commit:
- page loads without errors
- naming matches conventions
- imports are clean (@/...)
- no duplicate logic
- no console errors
- no unused files
- module matches template structure
- source files placed in correct pack category

RED FLAGS
- “I’ll just calculate this in the component”
- “I’ll quickly use this name for now”
- “I’ll grab this value directly from another module”
- “I’ll just use grid-cols-2 here”
- “I’ll fix naming later”

GOLDEN RULES
If unsure:
- Follow Labour pattern for input modules
- Follow Cost Summary pattern for summary pages
- Follow Cost Allocation pattern for structural pages

Do not invent new patterns.
