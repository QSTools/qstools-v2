QS Tools — Starter File Pack Template (v1.0 — LOCKED)

STATUS: COPY/PASTE BUILD STARTER

PURPOSE
This starter pack provides the base file templates for a new QS Tools module.

It is designed to:
- speed up new module builds
- keep naming consistent
- keep architecture separation clean
- reduce drift
- stop whole-page copy/paste chaos

REQUIRED FILE TREE
app/<module>/page.jsx
hooks/use<Module>.js
lib/calculations/<module>Calculations.js
lib/selectors/<module>Selectors.js
lib/storage/<module>Storage.js
lib/storage/<module>ProfileStorage.js
components/<module>/
  <Module>StatusStrip.jsx
  <Module>MainCard.jsx
  <Module>HelpPanel.jsx

Optional:
components/<module>/
  <Module>SummaryCard.jsx
  <Module>ProfilesCard.jsx
  <Module>Form.jsx
  <Module>InsightCard.jsx
  <Module>LinkTable.jsx

PAGE TEMPLATE
"use client";

import useModuleName from "@/hooks/useModuleName";
import ModuleNameStatusStrip from "@/components/module-name/ModuleNameStatusStrip";
import ModuleNameMainCard from "@/components/module-name/ModuleNameMainCard";
import ModuleNameHelpPanel from "@/components/module-name/ModuleNameHelpPanel";

export default function ModuleNamePage() {
  const { status, card } = useModuleName();

  return (
    <main className="ui-page">
      <div className="ui-page-stack">
        <ModuleNameStatusStrip {...status} />
        <ModuleNameMainCard {...card} />
        <ModuleNameHelpPanel />
      </div>
    </main>
  );
}

REPLACE LIST BEFORE FIRST USE
Replace all placeholders:
- ModuleName
- module-name
- useModuleName
- moduleStorage
- moduleProfileStorage
- moduleCalculations
- moduleSelectors
- example_input
- example_total

QUICK START RULE
1. choose module type
2. create file tree
3. paste starter files
4. rename placeholders
5. connect module brief contract
6. wire calculations
7. shape selectors
8. build UI only after contract is clear

NOT ALLOWED
- copying a whole existing page and editing blindly
- leaving old module names in files
- putting formulas in components
- bypassing hook/selectors
- using freestyle file names
- mixing module logic into another module
- styling outside the token/UI system
