## LABOUR COUNTING RULE

Cost Summary is allowed to count Labour profiles after the page is complete.

This count is derived from Labour data inside the selector layer.

### Required Labour-derived counts

* `saved_profile_count`
* `active_profile_count`

### Meanings

#### `saved_profile_count`

How many Labour profiles are stored in the Labour module data currently exposed to Cost Summary.

#### `active_profile_count`

How many of those stored Labour profiles are currently active and valid for Cost Summary use.

### Important distinction

* saved profiles = everything stored
* active profiles = records Cost Summary can actually use

Cost Summary calculations must use active Labour records only.

Status may use both saved and active counts.

## CALCULATION RESPONSIBILITY

`lib/calculations/costSummaryCalculations.js` owns all Cost Summary maths.

It must use the locked formulas from the Cost Summary module and page contracts.

### Locked formulas

#### Labour cost per productive hour

`labour_cost_per_productive_hour = labour_cost_rate + overhead_per_productive_hour`

#### Total labour cost annual

`total_labour_cost_annual = sum(labour_cost_per_productive_hour × productive_hours)`

#### Total productive output

`total_productive_output = sum(productive_hours)`

#### Total asset cost annual

`total_asset_cost_annual = sum(total_asset_cost_annual)`

#### Total business overheads

`total_business_overheads = total_general_overheads`

#### Total cost burden

`total_cost_burden = total_labour_cost_annual + total_asset_cost_annual + total_business_overheads`

#### Required revenue

`required_revenue = total_cost_burden`

#### Required recovery rate

`required_recovery_rate = required_revenue / total_productive_output`

### Rules

* no UI logic
* no approximation
* no scenario values
* no materials
* no subcontract logic
* no profit logic
* no budget logic
