# QS Tools — Master Project Brief (v1.10 FINAL — LOCKED)

---

## CORE OBJECTIVE

QS Tools is a **Commercial Decision Engine** designed to:

* determine true cost structures
* calculate required recovery
* compare actual vs required performance
* support safe, margin-driven commercial decisions

The system must prioritise **clarity, accuracy, and scalability** over speed or shortcuts.

---

## CORE PHILOSOPHY

* Break-even = risk
* Profit requires buffer (default target ≥ 20%)
* All outputs must be **derived from structured inputs**
* No manual “assumed” results
* Data must always be:

  * consistent
  * comparable
  * traceable

---

## 🔒 SYSTEM IDENTITY MODEL (CRITICAL)

QS Tools must use a **system-owned identity structure**.

### Identity Layers

* CompanyID (future — not implemented in V1)
* StaffID (mandatory — system generated)

### Structure

CompanyID
└── StaffID
  ├── Labour Profiles
  ├── Overhead Profiles
  ├── Time Logs (future)
  └── Cost Allocations (future)

---

## ⚠️ Identity Rules

* StaffID must be:

  * generated automatically (UUID or equivalent)
  * immutable
  * hidden from UI

* Staff Name must:

  * never be used as a key
  * only be used for display

* All modules must link using **StaffID only**

---

## 🔒 VARIABLE NAMING & DATA CONTRACT (CRITICAL)

All variable names form part of a **system-wide data contract**.

### Rules

* Variable names must be:

  * consistent
  * stable
  * globally reusable

* Once a variable is introduced and used:

  * it must **never be renamed**
  * it must **never be repurposed**

* All modules must use the same naming convention

---

### ⚠️ Breaking Rule

Changing a variable name is considered a:

👉 **breaking system change**

This requires:

* migration logic
* version control
* backward compatibility

---

### Naming Standard

* snake_case for all internal variables
* no spaces
* no abbreviations unless standardised
* names must reflect meaning, not UI labels

---

### Examples

Correct:

* staff_id
* base_rate
* total_cost_rate
* productive_hours

Incorrect:

* staffId
* costRate
* rate1
* temp_value

---

### Scope

This applies to:

* calculations
* selectors
* storage
* exports (SQLite)
* integrations
* APIs
* reporting (Power BI)

---

### Key Principle

👉 Variable names are part of the **system architecture**, not just code
