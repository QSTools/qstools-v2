# Cost Summary Page — Implementation Brief (v1.3 — LOCKED)

---

## 🎯 OBJECTIVE

Build the Cost Summary page as the internal cost + recovery decision engine of QS Tools.

It must:

- aggregate all internal cost components
- present structured, traceable cost breakdowns
- reflect the active recovery model
- calculate required recovery outputs
- provide decision-ready clarity

It must NOT:

- behave like a P&L
- show actual performance
- duplicate upstream calculations

---

## 🔒 SYSTEM ROLE

Cost Summary is the:

aggregation + recovery layer

Position:

Labour / Overheads / Assets / Allocation
                ↓
          COST SUMMARY
                ↓
             Budget

- productive_hours

### Employee Overheads
- overhead_per_productive_hour

### Assets
- total_asset_cost_annual

### General Overheads
- total_general_overheads

- active_revenue_model
- active_asset_labour_links[]
- linked_staff_count
- unlinked_active_staff_count

---

## 📊 CALCULATION RULES (LOCKED)

Must reuse:

labour_cost_per_productive_hour =
labour_cost_rate + overhead_per_productive_hour

total_labour_cost_annual =
sum(labour_cost_per_productive_hour × productive_hours)

total_productive_output =
sum(productive_hours)

total_cost_burden =
total_labour_cost_annual
+ total_asset_cost_annual
+ total_general_overheads

required_recovery_rate =
total_cost_burden / total_productive_output

All logic lives in calculation files only.

---

## ⚠️ CORE RULES

- MUST use upstream outputs only
- MUST NOT duplicate Labour logic
- MUST NOT include materials or subcontract logic
- MUST NOT become a P&L
- MUST NOT allow editing of recovery model
- MUST NOT rename variables
- MUST show warnings for incomplete data
- MUST keep identity-driven aggregation intact

---

## 🧱 UI RULES

- aggregated + drilldown structure REQUIRED
- expandable sections REQUIRED
- units must be clear (annual vs hourly)
- sorting must highlight largest contributors
- no cluttered multi-card layout
- no hidden assumptions

---

## 🚫 NOT ALLOWED

- performance metrics
- profit calculations
- forecast outputs
- margin vs actual
- editing Cost Allocation settings
- mixing scenario logic
- deep audit tables

## ✅ FINAL DEFINITION

The Cost Summary page is a structured aggregation and recovery layer that converts all internal cost inputs into a clear, traceable total cost burden and defines the required revenue and recovery rate needed to sustain the business.
