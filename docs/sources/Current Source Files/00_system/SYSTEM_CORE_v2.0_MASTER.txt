# QS Tools — System Core (v2.0 — MASTER)

STATUS
Active authority for system architecture and guardrails.

SYSTEM STACK

Input Modules
↓
Cost Allocation (structure only)
↓
Cost Summary (COST OF BUSINESS)
↓
Recovery Layer (future)
↓
Budget

CORE RULES

- Upstream modules own:
  - identities
  - inputs
  - calculations
  - output contracts

- Downstream modules:
  - consume output contracts only
  - must NOT rebuild upstream logic

MODULE ROLES

Labour → identity + labour cost engine  
Employee Overheads → staff overhead layer  
Assets → capital cost layer  
General Overheads → fixed business overhead layer  
Cost Allocation → structure only  
Cost Summary → cost-of-business layer  
Recovery Layer → interpretation (future)  
Budget → planning layer  

IDENTITY RULES

- staff_id → owned by Labour  
- asset_id → owned by Assets  
- joins must use IDs only  
- names are display only  

NAMING RULE

- snake_case only  
- no variable renaming without migration  
- contract names are locked  

SEPARATION RULE

- page = layout only  
- hook = orchestration  
- selectors = shaping  
- calculations = logic  
- components = UI only  

SCENARIO RULE

- Scenario is Labour-only  
- must NOT feed Cost Summary or Budget  

UI RULE

- token-based UI only  
- no inline styles  
- no hardcoded colours  
- mobile-first layout  

FINAL RULE

If a downstream module needs to “recalculate” upstream logic,
the system is already broken.
