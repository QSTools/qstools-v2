# QS Tools — Cost Allocation (v2.1 — LOCKED)

STATUS: STRUCTURAL + OPERATIONAL VALIDATION LAYER

---

## PURPOSE

The Cost Allocation Module validates whether the business structure can support the selected recovery strategy.

It defines:

- labour-to-asset relationships (capability)
- operational groups (delivery units)
- structural readiness for recovery

---

## SYSTEM ROLE (CRITICAL)

Cost Allocation is a:

structural + operational validation layer

Position:

INPUT MODULES
↓
COST SUMMARY
↓
RECOVERY SUMMARY
↓
COST ALLOCATION

---

## CORE PRINCIPLE

Recovery Summary defines WHAT must be recovered  
Cost Allocation defines WHETHER it can be delivered

## LAYER STRUCTURE

### Layer 1 — Capability Links

Defines:

what CAN work together

---

### Layer 2 — Operational Groups

Defines:

what MUST work together

Examples:

- Pump Unit = pump + ute + 1 driver  
- Drill Setup = digger + auger + 1 operator  

---

### Layer 3 — Validation

Checks:

- missing links
- orphan staff/assets
- incomplete groups
- structural coverage

---

## STRUCTURAL RULES

### labour_only

- links optional
- groups optional

---

### asset_driven

- links required
- groups required

---

### hybrid

- partial linking allowed
- warnings required

---

## VALIDATION RULES

### Duplicate Links

Not allowed:

same asset_id + staff_id active pair

---

### Orphan Detection

- staff not linked
- assets not linked
- assets not in groups

---

### Group Validity

Group is valid only if:

- all required assets exist
- required staff count is met

## DEFINITION

Cost Allocation is a structural and operational validation layer that ensures the business has the correct resource structure to support its recovery strategy without performing any cost or pricing calculations.
