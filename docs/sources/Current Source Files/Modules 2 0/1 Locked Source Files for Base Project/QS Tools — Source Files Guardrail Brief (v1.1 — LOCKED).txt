# QS Tools — Source Files Guardrail Brief (v1.1 — LOCKED)

PURPOSE

This brief defines the non-negotiable implementation rules for all future QS Tools development.

It exists to prevent:
- contract drift
- duplicated calculations
- module boundary violations
- incorrect recovery logic placement
- UI system drift
- architectural regression

This is a guardrail brief, not a feature brief.

## CORE SYSTEM SEPARATION

Cost Summary = WHAT it costs  
Recovery Summary = HOW it is recovered  
Cost Allocation = CAN it be delivered  

---

## UNIVERSAL DOWNSTREAM CONTRACT RULE

- Upstream modules own inputs and calculations
- Downstream modules consume outputs only
- No module may rebuild upstream maths
- No module may rename contract variables

---

## MODULE OWNERSHIP (UPDATED)

Labour
- owns staff_id
- owns labour cost engine

Employee Overheads
- owns staff-linked overheads

Assets
- owns asset_id
- owns asset cost engine

General Overheads
- owns business overhead totals

Cost Summary
- aggregates all upstream costs
- produces total_cost_burden

Recovery Summary
- defines recovery strategy
- distributes cost across labour/assets/overhead

Cost Allocation
- validates structure and operational readiness
- does NOT define recovery strategy

Budget
- consumes Cost Summary outputs only

---

## RECOVERY OWNERSHIP RULE (NEW)

- Recovery model is owned by Recovery Summary ONLY
- Cost Allocation must NOT define revenue_model
- Cost Allocation must consume active_recovery_model only

---

## IDENTITY RULE

- staff_id owned by Labour
- asset_id owned by Assets
- downstream modules must not recreate identities
- display names must never be used as keys

---

## PAGE / HOOK / CALCULATION SEPARATION

- page = orchestration only
- hook = integration
- calculations = business logic
- selectors = shaping
- components = UI only

No business logic allowed in UI components.

---

## SCENARIO ISOLATION RULE

- Scenario is labour-only
- must not feed Cost Summary
- must not feed Budget
- must not persist in V1

---

## UI SYSTEM RULE

- must use token-based UI system
- no inline styles
- no hardcoded colours
- must follow mobile-first layout rules

---

## NOT ALLOWED

- rebuilding upstream calculations
- defining recovery logic in Cost Allocation
- using display names as keys
- renaming contract variables
- placing business logic in UI
- mixing module responsibilities

---

## DEFINITION

This brief enforces the separation between cost, recovery, and structure layers to ensure QS Tools remains scalable, auditable, and commercially accurate.
