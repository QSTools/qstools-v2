Recovery Summary Page — Build Brief v1.0
Purpose

The Recovery Summary page is the new macro recovery strategy page.

It takes the already-aggregated business cost outputs from Cost Summary and answers:

what total cost must be recovered
how that recovery is expected to happen
how much must be recovered through labour
how much must be recovered through assets
how much remains absorbed overhead

It does not calculate upstream cost. Cost Summary already owns the total cost burden and required recovery baseline.

Module type

Summary Module. This follows the same summary-page pattern as Cost Summary.

System role

Recovery Summary is a commercial distribution layer.

It must:

consume Cost Summary outputs only
apply recovery strategy inputs
calculate recovery distribution outputs
expose clean recovery outputs downstream for Cost Allocation

It must not:

rebuild Labour maths
rebuild Asset maths
rebuild General Overheads maths
read raw upstream form state directly
become a pricing page

That downstream-contract behaviour matches the system guardrail.

Default behaviour

If no recovery settings exist yet:

recovery_model = labour_only
labour_share_percent = 100
asset_share_percent = 0
overhead_share_percent = 0

That default is important because in many builder/labour-only businesses the output may be effectively the same as Cost Summary, but Recovery Summary still provides the strategy explanation layer.

Core calculations
Share balance rule
labour_share_percent + asset_share_percent + overhead_share_percent = 100
Recovery cost allocation
labour_recovery_cost = total_cost_burden × labour_share_percent / 100
asset_recovery_cost = total_cost_burden × asset_share_percent / 100
overhead_absorbed_cost = total_cost_burden × overhead_share_percent / 100
Labour recovery rate
required_labour_recovery_rate = labour_recovery_cost / total_productive_output

If total_productive_output = 0:

required_labour_recovery_rate = 0
warning must be raised
Asset recovery requirement
required_asset_recovery = asset_recovery_cost

V1 keeps this annual and simple. No per-asset utilisation logic yet.

Wiring

Recovery Summary must wire from:

Cost Summary hook outputs only

It must not pull raw data from Labour, Assets, or General Overheads directly. That is exactly the downstream contract rule used in Cost Summary and reinforced by the guardrail brief.
