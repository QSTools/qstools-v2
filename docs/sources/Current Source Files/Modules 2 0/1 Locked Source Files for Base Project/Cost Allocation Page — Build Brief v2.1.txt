Cost Allocation Page — Build Brief v2.1
Purpose

The Cost Allocation page is the new micro structural and operational page.

It consumes:

Recovery Summary outputs for strategy
Labour active records for staff identity
Asset active records for asset identity

It answers:

does the current business structure support the selected recovery strategy
which staff and assets are linked
which assets/staff are unlinked
which minimum operational groups are valid or incomplete
Module type

Structural Module. This follows the existing Cost Allocation pattern, but aligned to the new Recovery layer and expanded with Operational Groups.

System role

Cost Allocation is a structural + operational validation layer.

It must:

consume Recovery Summary outputs
consume active Labour and Asset identities
define capability links
define operational groups
validate structural readiness

It must not:

calculate costs
calculate recovery rates
define pricing packages
own staff_id
own asset_id

That identity rule matches the Labour and Assets contracts.

Sequence
Cost Summary calculates the business cost burden and required baseline recovery.
Recovery Summary decides how that burden should be recovered.
Cost Allocation checks whether the actual operational structure can support that strategy.

That sequencing is cleaner than the current older baseline and avoids mixing macro recovery with micro resource structure. The current older Cost Allocation brief should be superseded by this aligned version.
