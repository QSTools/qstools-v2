# QS Tools — Recovery Summary Page (v1.0 — LOCKED)

STATUS: NEW LAYER (COMMERCIAL DISTRIBUTION)

---

## PURPOSE

The Recovery Summary Page defines how the total business cost burden is recovered.

It sits downstream of Cost Summary and converts total cost into a recovery strategy.

It answers:

- how much cost must be recovered
- how recovery is distributed (labour / assets / overhead)
- what minimum recovery pressure exists

---

## SYSTEM ROLE (CRITICAL)

Recovery Summary is a:

commercial distribution layer

Position:

INPUT MODULES
↓
COST SUMMARY
↓
RECOVERY SUMMARY
↓
COST ALLOCATION
↓
BUDGET (future)

---

## CORE PRINCIPLE

Cost Summary = WHAT the business costs  
Recovery Summary = HOW the business recovers it  

Recovery Summary must NOT:

- recalculate upstream costs
- consume raw module inputs
- become a pricing engine

It must:

- consume Cost Summary outputs only
- apply recovery strategy inputs
- produce recovery outputs for downstream use

### Local Inputs

- recovery_model

Allowed values:

- labour_only
- asset_driven
- hybrid

- labour_share_percent
- asset_share_percent
- overhead_share_percent

---

## DEFAULT BEHAVIOUR

If no inputs:

recovery_model = labour_only  
labour_share_percent = 100  
asset_share_percent = 0  
overhead_share_percent = 0  

---

## CALCULATIONS

### Share Balance Rule

labour_share_percent + asset_share_percent + overhead_share_percent = 100

---

### Recovery Distribution

labour_recovery_cost =
total_cost_burden × labour_share_percent / 100

asset_recovery_cost =
total_cost_burden × asset_share_percent / 100

overhead_absorbed_cost =
total_cost_burden × overhead_share_percent / 100

---

### Labour Recovery Rate

required_labour_recovery_rate =
labour_recovery_cost / total_productive_output

---

### Asset Recovery Requirement

required_asset_recovery =
asset_recovery_cost
