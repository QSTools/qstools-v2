QS Tools — Rate Models Module (v1.0 — LOCKED)
PURPOSE

The Rate Models Module defines how the business converts operating resources into sellable production outputs.

This includes rate structures such as:

square metre rates
tonne rates
cubic metre rates
minimum charges
small-job uplift logic
large-volume rate efficiency logic
SYSTEM ROLE (CRITICAL)

The Rate Models Module is a:

production-based revenue model layer

It must:

define sellable units
define rate structures
support minimum charges
support volume-sensitive rate logic
expose expected recovery outputs

It must NOT:

replace Cost Summary
replace Materials
replace Cost Allocation
become a job estimating engine in v1.0
ARCHITECTURE

Page shell only.
Sub-pages own the actual model logic.

This module groups:

Square Metre Rate Page
Volume Rate Page
CORE PRINCIPLE

This module answers:

“What do we physically produce and at what rate do we sell it?”

Examples:

m² placed
tonnes moved
m³ supplied and installed
OUTPUT CONTRACT RULE

Every rate page must expose:

rate_model_type
unit_label
minimum_charge
base_rate
expected_annual_volume
expected_revenue_annual
expected_recovery_contribution
warnings[]
DEFINITION

The Rate Models Module is the production-output layer of QS Tools. It converts real-world operating activity into sellable units and rate-based revenue assumptions so those outputs can later be tested against total business cost.
