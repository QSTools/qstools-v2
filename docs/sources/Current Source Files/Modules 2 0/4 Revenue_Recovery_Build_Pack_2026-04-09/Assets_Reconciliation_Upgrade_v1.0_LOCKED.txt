# QS Tools — Assets Reconciliation Upgrade (v1.0 — LOCKED)

STATUS: ASSET BENCHMARK CHECK LAYER

---

## PURPOSE

This brief upgrades the Assets page to include a reconciliation strip comparing the assigned P&L asset benchmark against the QS Tools asset model total.

It exists to:

- check whether the asset model explains the assigned accounting asset cost benchmark
- surface missing asset costs clearly
- confirm readiness for trusted modelling

---

## SYSTEM ROLE (CRITICAL)

Assets remains the standalone capital cost engine of QS Tools.

The reconciliation strip is:

a benchmark visibility + readiness layer only

The Assets module must continue to own:

- asset identity
- asset cost calculations
- asset outputs used downstream

---

## FILES TO UPDATE

- hooks/useAssets.js
- lib/selectors/assetSelectors.js
- components/assets/AssetStatusStrip.jsx
- components/assets/AssetHelpPanel.jsx

Reusable inputs come from:
- reconciliationCalculations.js
- ReconciliationStrip.jsx

---

## NEW INPUT

From P&L grouped outputs:

- assets_benchmark_total

---

## THRESHOLDS

### Green
variance_percent <= 0.5

### Amber
variance_percent > 0.5 and <= 1.0

### Red
variance_percent > 1.0

---

## READINESS RULE

Green = ready
Amber = not fully ready
Red = blocked

---

## HELP PANEL UPDATE

AssetHelpPanel must explain:

- benchmark comes from assigned asset-related P&L lines
- model total comes from QS asset inputs
- large variance usually means missing asset costs or incorrect assignment

---

## CORE RULES

- benchmark is reference only
- asset model total remains downstream source
- variance does not flow into Cost Summary formulas

---

## DEFINITION

This upgrade adds an asset reconciliation layer to the Assets page so the QS asset model can be checked against the assigned accounting benchmark before trusted downstream modelling proceeds.
