CLARIFICATION

The P&L Page is now the official user entry point.

It captures:

- annual business performance
- accounting-style revenue structure
- accounting-style cost structure
- actual profit outcome

It does NOT replace:

- Cost Summary
- Recovery Summary
- Materials
- Rate Models
- Cost Allocation
- Recovery Outcome

P&L remains a truth-input layer only.

P&L ROLE CLARITY (LOCKED)

P&L is a parallel truth layer.

It provides:

- total_trading_income
- total_cost_of_sales
- gross_profit
- total_other_income
- total_operating_expenses
- net_profit

It is used to help validate the model at the outcome layer.

It must NOT:

- drive upstream calculations
- alter Cost Summary
- alter Recovery Summary
- alter Materials
- alter Rate Models

RULE ENFORCEMENT

No module may rebuild upstream maths  
No module may rename contract variables  
Cost Summary remains unchanged  
Recovery logic must be derived, not manual  
P&L must remain a truth-input layer only  
Materials must remain a material margin layer only  
Rate Models must remain production-based revenue layers only
