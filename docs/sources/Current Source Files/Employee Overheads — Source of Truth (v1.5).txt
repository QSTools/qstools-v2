# Employee Overheads тАФ Source of Truth (v1.5)

## Purpose

The Employee Overheads Module captures all non-wage staff overhead costs and converts them into an hourly burden linked to an existing Labour profile.

This module sits on top of Labour.

It does not create staff identity.
It does not calculate base labour cost.
It only applies staff-specific overhead burden to a staff member already created in the Labour Module.

---

## ЁЯФТ SYSTEM ROLE (CRITICAL)

The Employee Overheads Module is a:

linked burden layer

It must always attach to a valid Labour profile using:

* `staff_id` (hidden, system-linked)

Identity is owned by the Labour Module.

Employee Overheads must never own, create, or control staff identity.

## тЪая╕П IDENTITY RULES

* `staff_id` is required for every overhead profile
* `staff_id` must be hidden from the UI
* `staff_name` is display only
* `staff_name` must never be used as a key
* Employee Overheads must not allow manual creation of staff identity
* Employee Overheads must not edit Labour-owned identity fields

This follows the system identity model where Labour owns StaffID and all linked modules use that identity only.

---

## тЪая╕П VARIABLE NAMING RULE

All internal variables in this module form part of the system-wide data contract.

Rules:

* use `snake_case` only
* variable names must remain stable
* variables must never be renamed without migration
* variables must reflect meaning, not UI wording
* variables must remain reusable across selectors, storage, exports, integrations, and reporting

This module must follow the locked naming rules defined by the master brief.

## ЁЯУК Outputs

### `total_annual_overheads`

Sum of all annual overhead input fields

### `overhead_per_paid_hour`

`total_annual_overheads / paid_hours_per_year`

### `overhead_per_productive_hour`

`total_annual_overheads / productive_hours`

### `labour_cost_per_productive_hour`

`labour_cost_rate + overhead_per_productive_hour`

## ЁЯФЧ Labour Dependency (MANDATORY)

This module depends on Labour for:

* `staff_id`
* `staff_name`
* `staff_role`
* `labour_class`
* `paid_hours_per_year`
* `productive_hours`
* `labour_cost_rate`

Employee Overheads must not duplicate Labour calculations.

It must only:

* consume Labour outputs
* apply staff-specific overhead burden on top
* produce linked commercial recovery outputs

This aligns with the rule that overheads sit on top of labour and remain modular.

## ЁЯФБ Versioning Rules

Each staff member must support:

* one active overhead profile
* historical overhead profile versions

Rules:

* editing creates a new version
* only one active version may exist per `staff_id`
* historical versions must remain unchanged
* summary logic must use the active version only

---

## ЁЯФЧ Integration Rules

### Labour Module

* required source module
* owns identity
* owns labour cost logic
* owns paid and productive hour logic

### Cost Summary Module

* consumes active labour result plus active overhead result
* must link by `staff_id` only

### Budget Module

* may use overhead burden for forecasting and future planning

### Future Accounting Integration

* may suggest or prefill categories from external accounting systems
* all imported values must still be normalised into internal variable names before use or storage

## ЁЯз▒ Design Principles
