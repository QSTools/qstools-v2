# QS Tools — Model Readiness (v3.5 — LOCKED)

STATUS: TRUST GATE

---

## PURPOSE

Model Readiness determines whether the cost model is complete enough to trust downstream recovery, pricing, capacity, and validation outputs.

---

## CORE PRINCIPLE

If the model is not ready, downstream pages are not trusted.

---

## INPUTS

- pnl_ready
- labour_ready
- assets_ready
- general_overheads_ready

---

## REMOVED INPUT

- employee_overheads_ready

---

## CALCULATION

model_ready =
pnl_ready
AND labour_ready
AND assets_ready
AND general_overheads_ready

---

## OUTPUTS

- model_ready
- blocking_modules
- warning_modules

---

## BEHAVIOUR

If model_ready = false:

- Cost Summary totals may display
- downstream modelling is marked not trusted
- Recovery Summary / Rate Models / Production Capacity / Recovery Outcome should not be treated as final

Model Readiness v3.5 is the trust gate for QS Tools. It confirms P&L, Labour, Assets, and General Overheads are reconciled before downstream modelling is trusted.
