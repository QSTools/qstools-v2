QS Tools — UI Implementation Standard (v1.1 — LOCKED)

---

PURPOSE

This brief defines the enforced UI implementation rules for all QS Tools pages.

It exists to:

* standardise page structure across the system
* ensure all pages use the shared UI layer
* prevent UI drift and rework
* allow future visual changes via globals.css only

---

CORE PRINCIPLE

Pages must use the shared UI system.

No page owns its own styling system.

---

SHARED UI SYSTEM (LOCKED)

All pages must use the following classes:

Page shell

* ui-page
* ui-page-stack

Sections

* ui-section
* ui-section-muted
* ui-panel

Inputs

* ui-input
* ui-select
* ui-textarea
* ui-readonly

Actions

* ui-button-primary
* ui-button-secondary
* ui-button-danger

Text

* ui-display
* ui-lead
* ui-kicker
* ui-label
* ui-help

Layout + spacing

* ui-stack
* ui-stack-sm
* ui-actions
* ui-split
* ui-split-2

Utility

* ui-pill

---

APPROVED SPECIAL LAYOUTS

These remain valid where already defined:

* labour-layout
* cost-summary-layout

No additional layout systems should be created without a new locked brief.

---

PAGE STRUCTURE (MANDATORY)

All pages must follow:

<main className="ui-page">
  <div className="ui-page-stack">
    ...
  </div>
</main>

---

RESPONSIVE RULE (LOCKED)

Mobile (base)

* single column
* vertical stacking
* 44px minimum controls

Tablet

* maximum two-column layouts
* controlled use of ui-split-2

Desktop

* supporting rails allowed
* use approved layout helpers only
* no uncontrolled grid systems

---

COMPONENT RULES

Sections
Use ui-section for all primary page blocks.

Panels
Use ui-panel for nested groupings.

Inputs
Must use shared UI input classes.

Buttons
Must use shared button classes only.

---

SPACING RULE

Use only:

* ui-page-stack
* ui-stack
* ui-stack-sm
* ui-actions

Do not manually construct spacing systems inside pages.

---

NOT ALLOWED

* inline styles for core layout or design
* page-specific design systems
* custom colour tokens
* Tailwind utility styling as the primary design system
* duplicate button systems
* duplicate card systems
* ad hoc responsive grids

---

SYSTEM SEPARATION (CRITICAL)

This standard applies to UI only.

It must NOT affect:

* calculations
* selectors
* hooks
* storage
* data contracts
* module ownership

---

FUTURE DESIGN CHANGES RULE

All visual updates must be applied through globals.css.

Pages must not require updates for visual redesigns.

---

SUCCESS CRITERIA

* all pages use the shared UI system
* no duplicate styling systems exist
* responsive behaviour is consistent
* pages do not require styling rework during redesign

---

FINAL DEFINITION

This standard locks the shared UI system as the only allowed presentation layer for QS Tools pages.
