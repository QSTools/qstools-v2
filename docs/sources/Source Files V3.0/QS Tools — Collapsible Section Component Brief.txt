# QS Tools — Collapsible Section Component Brief

### v1.0 — LOCKED UI PATTERN

STATUS: READY FOR IMPLEMENTATION

---

## PURPOSE

Define the **standard collapsible section component** used across QS Tools.

This component improves:

* scanability
* usability
* consistency across modules

---

## CORE PRINCIPLE

Each grouped section must:

* remain useful when collapsed
* show its total at all times

---

## HEADER CONTENT (LOCKED)

Each section header must show:

* section title
* live subtotal
* expand/collapse control

Example:

```text
Office / Admin              $45,000     ▼
```

---

## BEHAVIOUR

### Collapsed

* title visible
* subtotal visible
* content hidden

### Expanded

* title visible
* subtotal visible
* content visible

---

## PROPS (LOCKED)

```javascript
{
  title: string,
  subtotal_value: number,
  is_open: boolean,
  on_toggle: function,
  children: ReactNode
}
```

---

## RULES

* subtotal is passed in (not calculated inside component)
* no business logic inside component
* reusable across all modules

---

## WHERE TO USE

* P&L grouped sections
* General Overheads categories
* any grouped financial inputs

---

## DO NOT USE

* summary cards
* help panels
* non-financial sections

---

## SUCCESS CRITERIA

* subtotal visible when collapsed
* consistent behaviour across modules
* no duplicated UI logic

---

END
