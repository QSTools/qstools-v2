# QS Tools — UI Migration Plan (v1.0 — LOCKED)

STATUS: IMPLEMENTATION PLAN — SAFE MIGRATION
---

## PURPOSE

This brief defines how the new demo UI system will be rolled into the main QS Tools application without breaking:

* core calculations
* module logic
* layout structure
* system integrity

---

## CORE PRINCIPLE

> Do not rebuild the system — restyle and restructure it using the shared UI layer.

---

## NON-NEGOTIABLE RULES

1. No module creates its own colours
2. No module overrides theme tokens
3. All styling must use shared CSS system (globals.css)
4. No changes to calculation logic during UI migration
5. No changes to data contracts
6. No breaking changes to layout shell (RootLayout + Sidebar)

---

## SYSTEM LAYERS (LOCKED)

### Layer 1 — Shell (DO NOT CHANGE)

* RootLayout
* SidebarNavigation
* Routing structure

---

### Layer 2 — Theme System (EXTEND ONLY)

* globals.css
* ui-* component classes
* theme variables

Allowed:

* add missing utility classes
* extend spacing and layout helpers

Not allowed:

* rewriting theme system
* introducing new colour systems

---

### Layer 3 — Page Structure (PRIMARY CHANGE AREA)

Pages will adopt:

* clearer section hierarchy
* consistent card structure
* improved spacing
* improved readability

---

### Layer 4 — Components (CONTROLLED UPDATE)

* migrate to ui-* system
* align inputs / outputs
* standardise layout patterns

---

## TARGET UI PATTERNS

### 1. Page Structure

Every page should follow:

* Page wrapper (ui-page)
* Page stack (ui-page-stack)
* Section blocks (ui-section)
* Panel blocks (ui-panel)

---

### 2. Interaction Pattern

For interactive pages:

* LEFT = inputs
* RIGHT = outputs

Pattern must match Quick Start structure

---

### 3. Card Hierarchy

* Section = major grouping
* Panel = content block
* Inputs = ui-input
* Actions = ui-actions

---

### 4. Language Style

Use:

* simple
* direct
* non-technical

Avoid:

* financial jargon
* complex phrasing

---

* mobile-first layout
* stack on small screens
* split layout on desktop
* use existing grid + media queries

---

## TESTING CHECKLIST

After each page migration:

* layout does not break
* sidebar remains stable
* no overflow issues
* inputs still function
* calculations unchanged
* responsive behaviour correct

---

## SUCCESS CRITERIA

* consistent UI across all pages
* no duplicated styling logic
* no module-specific design systems
* improved readability
* faster user understanding

---

## FINAL DEFINITION

This rollout migrates QS Tools to a unified UI system by applying a consistent layout and component structure across all pages, without modifying business logic or system architecture.
