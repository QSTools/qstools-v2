# QS Tools — Full Platform Architecture (v1.0 — LOCKED)

STATUS: PRODUCT STRUCTURE + SYSTEM FLOW

---

## PURPOSE

This brief defines the full QS Tools platform structure and how each layer connects.

It exists to:

* align product development
* prevent feature drift
* ensure consistency across demo and full system
* define what QS Tools actually is

---

## CORE PRINCIPLE

> QS Tools is a business alignment system — not a calculator and not just a pricing tool.

---

## PLATFORM STRUCTURE

QS Tools consists of three connected layers:

---

## 🔹 LAYER 1 — DEMO (QUICK START)

### PURPOSE

Create awareness and trigger realisation.

---

### FUNCTION

* expose gaps between perceived and actual performance
* give a fast, low-friction entry point
* demonstrate the value of the system

---

### INCLUDES

* Labour Reality Check
* Square Metre Calculator
* Materials + Labour Calculator

---

### OUTPUT

* highlights mismatch between rate and reality
* creates “this might not be working” moment

---

### DOES NOT

* solve the problem
* model the business
* define correct pricing

---

### ROLE IN SYSTEM

> Entry point into QS Tools

---

## 🔹 LAYER 2 — CORE SYSTEM

### PURPOSE

Model the business correctly.

---

### FUNCTION

* define full cost structure
* define recovery requirements
* define business baseline

---

### INCLUDES

* P&L Page
* Labour Module
* Employee Overheads
* Assets
* General Overheads
* Cost Summary
* Business Model
* Recovery Summary

---

### OUTPUT

* required revenue
* required recovery structure
* baseline business model

---

### ROLE IN SYSTEM

> Defines what the business must achieve to work

---

## 🔹 LAYER 3 — QUOTE / EXECUTION

### PURPOSE

Apply the business model to real work.

---

### FUNCTION

* evaluate jobs and quotes
* compare against baseline
* validate decisions

---

### INCLUDES

* Rate Models
* Cost Allocation
* Quote Benchmark
* Recovery Outcome

---

### OUTPUT

* whether a job works
* whether pricing aligns
* whether recovery is achieved

---

### ROLE IN SYSTEM

> Tests real-world decisions against the model

---

## FULL SYSTEM LOOP

1. Build business model (Layer 2)
2. Define required recovery
3. Input job / quote (Layer 3)
4. Validate outcome

---

## DEMO CONNECTION

The Demo Layer connects directly into the Core System.

Flow:

Demo → Awareness
Core System → Understanding
Execution → Validation

---

## PRODUCT DEFINITION

QS Tools is:

* a system that models real business cost
* a system that defines required recovery
* a system that validates pricing and decisions

QS Tools is NOT:

* a simple calculator
* a generic pricing tool
* a one-step quoting solution

---

## USER JOURNEY

1. User enters through Demo
2. User recognises potential issue
3. User builds full business model
4. User validates real work

---

## KEY DIFFERENTIATOR

> QS Tools does not tell users what to charge — it shows whether their business works.

---

## CORE VALUE

* clarity
* alignment
* confidence in decisions

---

## SYSTEM RULE

Each layer must remain separate:

* Demo must not contain full system logic
* Core System must not be simplified into demo logic
* Execution must not redefine the model

---

## DEFINITION

QS Tools is a structured system that allows businesses to model their cost base, define recovery requirements, and validate whether their pricing and real-world work align with that reality.

---

## FINAL LINE

> QS Tools mirrors your business, defines your baseline, and tells you whether your decisions actually work.
