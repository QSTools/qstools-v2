# QS Tools — General Overheads Amendment (v2.3 — Phase 1 Role Update).txt"

### v2.3 — FINAL STRUCTURE

STATUS: READY FOR BUILD

---

## PURPOSE

Convert P&L expense data into a **clean, structured overhead model**.

---

## CORE PRINCIPLE

P&L = what you spent
General Overheads = what it costs to run the business

---

## ROLE

* review P&L-fed data
* reclassify items
* remove non-overhead costs
* produce final overhead total

---

## CATEGORIES (LOCKED)

* Office / Admin
* Financial Admin
* Insurance & Compliance
* Vehicles (Running)
* Staff Overheads
* Facilities / Premises
* Sales / Growth
* Other / Unallocated

---

## EXCLUSIONS (CRITICAL)

Remove:

* wages
* PAYE / ESCT / KiwiSaver
* labour-related inputs
* asset finance / ownership

---

## INPUT BEHAVIOUR

* auto-populate from P&L
* user adjusts (not re-enters)

---

## RECLASSIFICATION

### Desktop / Tablet

* drag and drop

### Mobile

* dropdown selector

---

## CATEGORY UI

Each category must:

* be a collapsible section
* show subtotal in header
* expand to show items

---

## SUMMARY CARD (LOCKED)

### A. Total

```text
Total General Overheads: $XXX
```

---

### B. Breakdown (with %)

```text
Category   $X   (X%)
```

Rule:

```text
% = category_total / total_general_overheads
```

---

### C. Reconciliation

```text
P&L Overheads (excl. Labour & Assets): $XXX
Model Overheads: $XXX
Variance: $XXX (+/- X%)
Status: Aligned / Needs Review
```

---

## OUTPUT CONTRACT

```javascript
{
  total_general_overheads: number,
  category_totals: object,
  benchmark_total: number,
  variance: number,
  ready: boolean
}
```

---

## GOAL

User can clearly see:

* total overhead cost
* where money is going
* whether it aligns with P&L
