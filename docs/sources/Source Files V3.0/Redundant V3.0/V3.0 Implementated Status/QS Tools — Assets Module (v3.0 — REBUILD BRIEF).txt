# QS Tools — Assets Module (v3.0 — REBUILD BRIEF)

STATUS: REBUILD REQUIRED (ALIGN TO SIMPLIFIED MODEL + RECONCILIATION LAYER)

---

## PURPOSE

The Assets Module defines the annual cost of owned business assets.

It is responsible for:

* capturing per-asset cost inputs
* calculating annual asset cost
* producing clean asset cost outputs for downstream consumption
* validating asset cost completeness against a P&L benchmark

It defines:

**WHAT assets cost the business annually**

---

## SYSTEM ROLE (CRITICAL)

Assets is an:

**input module (asset cost engine)**

Position:

INPUT MODULES
↓
COST SUMMARY
↓
RECOVERY SUMMARY
↓
COST ALLOCATION

Assets owns asset cost only. It must remain separate from recovery and structural logic.

---

## CORE PRINCIPLE

Assets must:

* calculate per-asset cost only
* aggregate total asset cost
* remain simple and input-driven
* provide clean downstream outputs

Assets must NOT:

* allocate costs across assets
* distribute pooled costs
* link assets to staff
* calculate recovery or pricing
* simulate utilisation or revenue
* consume Scenario outputs

This matches the active system ownership rules.

---

## CRITICAL CHANGE (THIS REBUILD)

### REMOVE SHARED COST DISTRIBUTION LOGIC

The Phase 1 amendment removes pooled/shared cost distribution inside Assets.

Do NOT build or preserve logic for:

* fuel split across vehicles
* shared maintenance allocation
* pooled servicing distribution
* fleet-level distribution logic

These are excluded from Assets in Phase 1 and belong outside this module.

---

## IMPORTANT NAMING / WIRING RULE

This rebuild must **not rename locked variables or existing wired field names** unless a full migration is being done across storage, selectors, hook outputs, components, and downstream consumers. Variable stability is a system rule.

### Therefore, KEEP existing output names:

* `finance_cost_annual`
* `running_cost_annual`
* `total_asset_cost_annual`

These are already exposed in the current calculations and hook output contract.

### KEEP existing current field names for now:

* `maintenance_cost_monthly`
* `fuel_cost_monthly`
* `registration_cost_monthly`
* `other_running_cost_monthly`

These may remain in this rebuild **to preserve wiring**, but their use must be controlled by the updated behaviour rule below.

---

## UPDATED BEHAVIOUR RULE

Existing running-cost inputs may remain technically present, but they must now be treated as **direct per-asset costs only**.

That means:

* valid only where the cost is clearly attributable to that single asset
* not to be used for pooled/shared fleet allocation
* not to be used for business-wide fleet assumptions
* not to be distributed across multiple assets inside this module

So the field names stay, but the module behaviour is simplified and corrected. This aligns the rebuild with the simplification amendment without breaking wiring.

## NEW P&L RECONCILIATION LAYER

This rebuild must add a reconciliation layer using a benchmark reference.

### New input

* `assets_benchmark_total`

### New outputs

* `assets_variance_amount`
* `assets_variance_percent`
* `assets_ready`

### Thresholds

* Green: variance percent `<= 0.5`
* Amber: variance percent `> 0.5 and <= 1.0`
* Red: variance percent `> 1.0`

### Readiness rule

* Green = ready
* Amber = not fully ready
* Red = blocked

This reconciliation update is part of the locked Assets direction.
