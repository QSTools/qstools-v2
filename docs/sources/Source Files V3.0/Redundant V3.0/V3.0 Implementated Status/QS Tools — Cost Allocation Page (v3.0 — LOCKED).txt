# QS Tools — Cost Allocation Page (v3.0 — LOCKED)

STATUS: CONSTRAINT / DELIVERY CHECK LAYER

---

## PURPOSE

The Cost Allocation Page identifies what limits the business from delivering its required output.

---

## CORE PRINCIPLE

> Only model what limits output — ignore everything else

---

## INPUTS

From Rate Models:

* required_output_per_period

From Labour:

* available_labour_hours

From Assets (owned only):

* key constraint assets (if applicable)

---

## CALCULATION

capacity_available =
based on labour + time + key constraints

---

capacity_required =
from Rate Model output

---

capacity_gap =
capacity_available - capacity_required

---

## OUTPUTS

* capacity_available
* capacity_required
* capacity_gap
* constraint_flags

---

## WHAT THIS PAGE DOES

Identifies:

* labour shortages
* time constraints
* key bottlenecks
* delivery limits

---

## WHAT THIS PAGE DOES NOT DO

* calculate rates ❌
* calculate cost ❌
* optimise structure ❌
* track asset utilisation ❌

---

## RULE

> If it does not limit output, it does not belong here

---

## DEFINITION

Cost Allocation v3.0 is the constraint layer of QS Tools. It identifies whether the business has the capacity to deliver the required output and highlights what will stop it from doing so.
