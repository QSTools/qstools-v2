# QS Tools — P&L Page Update Brief

### v1.1 — GROUPED + COLLAPSIBLE UPDATE

STATUS: READY FOR UPDATE

---

## PURPOSE

Improve the P&L page by:

* grouping categories
* improving readability
* adding collapsible sections with subtotals

---

## CORE PRINCIPLE

P&L remains:

👉 input + classification layer

NOT:

* filtered model
* overhead engine

---

## UPDATE REQUIREMENTS

### 1. Apply grouped categories

Use same structure:

* Office / Admin
* Financial Admin
* Insurance & Compliance
* Vehicles (Running)
* Staff Overheads
* Facilities / Premises
* Sales / Growth
* Other / Unallocated

---

### 2. Add collapsible sections

Each section must:

* be collapsible
* show subtotal in header
* use shared component

Example:

```text
Office / Admin              $45,000     ▼
```

---

### 3. Subtotal behaviour

* always visible
* updates live
* derived from grouped values

---

## WHAT THIS DOES NOT CHANGE

* no filtering of labour
* no removal of asset costs
* no downstream logic changes

---

## USER EXPERIENCE

User should be able to:

* scan entire P&L at a glance
* see grouped totals instantly
* open sections only when needed

---

## SUCCESS CRITERIA

* grouped sections implemented
* subtotals visible when collapsed
* page easier to scan
* aligned visually with General Overheads
* no logic duplication

---

## DEFINITION

This update improves the usability of the P&L page by introducing grouped collapsible sections with visible subtotals while preserving its role as the primary input layer.
