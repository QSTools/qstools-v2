# QS Tools — P&L Input Setup Flow Completion Brief (v3.2)

STATUS: COMPLETE / READY TO APPLY PATTERN DOWNSTREAM

## PURPOSE

This brief captures the completed P&L page rebuild and defines the pattern to reuse across Labour, Assets, General Overheads, and future setup modules.

The P&L page is the first setup step.

It exists to:

- capture the actual business P&L
- classify costs into QS Tools setup buckets
- create P&L benchmark totals
- guide the user into the next module

The page is not an analysis page.

It is the start of the first-principles business model setup.

---

## CORE CONCEPT

P&L = what happened.

QS Tools modules = first-principles rebuild of how the business operates.

Reconciliation = whether the model is close enough to reality.

The P&L page does not replace Labour, Assets, or Overheads.

It creates the benchmark view that later modules compare against.

---

## FINAL USER FLOW

1. User enters real P&L
2. User classifies operating expenses
3. User reviews P&L Summary
4. User resolves warnings
5. User saves P&L
6. User continues to General Overheads

Final flow:

Enter → Classify → Review → Save → Continue

- Financial Admin

- Facilities / Premises
- Sales / Growth
- Other / Unallocated

---

## P&L DEFAULT LINE CLEANUP

Default P&L lines now better match practical business categories.

Key changes:

- Fuel added as a default operating expense line
- Fuel belongs to Assets
- Licences & Registrations belongs to Assets
- Repairs & Maintenance belongs to Assets
- KiwiSaver Employer Contributions belongs to Labour
- ACC Levy belongs to Labour
- Tools & Equipment belongs to Employee Overheads
- Travel / Entertainment / Penalties are treated as Other / Unallocated for review

## QS BENCHMARK VIEW

The P&L Summary includes a QS Benchmark View.

Purpose:

These are the P&L benchmark totals used to check Labour, Assets, and Overheads later.

Benchmark totals include:

- Revenue
- COGS - Materials
- COGS - Subcontract
- COGS - Hire
- Total COGS
- Labour Benchmark
- Employee Overheads Benchmark
- Assets Benchmark
- General Overheads Benchmark
- Total Business Costs

---

## ASSIGNMENT CHECK

Assignment Check now lives inside P&L Summary.

It shows:

- Total Business Costs
- Total Assigned
- Unassigned Balance
- P&L Ready

This makes the user review classification before saving and continuing.

---

## WARNINGS

Warnings now live inside P&L Summary, before Save P&L.

Warnings use the danger theme.

Warnings are visible and strong, but do not block save.

Examples:

- No trading income has been entered yet.
- No cost of sales has been entered yet.
- Some business costs are still unassigned.
- Gross profit is negative.

---

## SAVE AND CONTINUE

Save action has moved from the page header into the P&L Summary.

Final actions:

- Save P&L
- Next: Overheads

The user saves after reviewing the summary, assignment check, and warnings.

---

## WHAT WAS REMOVED

The following were removed from the top-level page flow:

- Status Snapshot card
- Reconciliation strip
- Separate Assignment Check panel
- Save Snapshot wording

Reason:

These created duplication and noise.

The P&L Summary is now the single review point.

---

## UI RULES

- All panels collapse by default
- Entire page should be visible without scrolling on normal desktop view
- No duplicated summary/status panels
- No interpretation during input
- Warnings appear only at review point
- Save and Next actions live at the end of the setup step

---

## SYSTEM RULES

P&L owns:

- P&L line capture
- P&L category classification
- benchmark totals

P&L does not own:

- Labour calculations
- Asset calculations
- General Overhead calculations
- Cost Summary calculations

Downstream modules must compare against P&L benchmarks, not rebuild P&L logic.

---

## NEXT MODULE PATTERN

Every setup module should follow:

1. Page title / purpose
2. Inputs
3. Summary
4. Reconciliation / readiness check
5. Warnings
6. Save
7. Next module button

For Labour:

- simple staff / hours / rate input
- auto-calculated first-principles labour baseline
- reconciliation against P&L Labour Benchmark
- Next: Assets or Next: Overheads depending final setup order

For Assets:

- asset baseline model
- reconciliation against P&L Assets Benchmark
- Next module button

For General Overheads:

- finalise overhead categories
- reconcile against P&L General Overheads Benchmark
- Next: Labour

---

## FINAL DEFINITION

The P&L page is the first setup layer in QS Tools.

It captures what happened, classifies it, creates benchmark totals, and guides the user into the first-principles rebuild of the business.
