QS Tools — Labour Page UI Cleanup & Layout Alignment (v2.5 — FINAL)

STATUS
IMPLEMENTED
LOCKED BASELINE

OBJECTIVE
Refactor the Labour page to:
- remove duplicated calculation displays from input cards
- simplify the right-side output card to live rate outputs only
- improve readability and flow of the page
- align layout with the Phase 4 responsive layout contract
- reinforce system architecture separation (inputs → outputs → explanation → decision)

CORE INTENT
The Labour page must follow this fixed structure:
- Input cards → data entry only
- Rate Builder (right panel) → live Labour answer
- Labour Summary → explanation layer
- Top Driver → decision layer

The same value must not appear in multiple places.

NON-GOALS (LOCKED)
This phase does not:
- include employee overheads in Labour
- include asset costs
- include general overhead allocation
- represent full business recovery
- replace Cost Summary or Recovery Summary layers

Labour remains a Labour-only model layer.

LABOUR LIMITATION (CRITICAL)
The Labour module produces a Labour-only recovery position.

It does not include:
- employee overhead allocation
- business overhead recovery
- asset utilisation recovery

Therefore:
The charge-out rate shown is not the final required business rate.

It is the Labour component of recovery only.
Full recovery is completed in downstream modules.

USER MENTAL MODEL (LOCKED)
1. Enter inputs
2. Observe live Labour rate (right panel)
3. Understand how it is built (summary)
4. Identify main pressure (top driver)
5. Adjust inputs

This flow must be preserved in all future updates.

INPUT CARD CLEANUP (CRITICAL)
Rule:
Input cards are input-only and must not display derived outputs.

Cards updated:

WorkingPatternCard
- keep: hours_per_week, days_per_week
- remove: paid hour outputs

PayCard
- keep: labour_rate, charge_out_rate
- remove: all calculated outputs

CommercialCard
- keep:
  - productivity_percent
  - margin_target_percent
- remove: all rate/cost outputs

EntitlementsCard
- keep: entitlement inputs
- remove: non-productive hour outputs

EmployerContributionsCard
- keep: KiwiSaver inputs
- remove:
  - employer contribution outputs
  - ESCT outputs

Result:
- eliminates duplication
- improves clarity
- enforces structure
