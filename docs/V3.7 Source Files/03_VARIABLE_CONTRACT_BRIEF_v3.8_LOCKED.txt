# QS Tools — Variable Contract Brief

## v3.8 — MODE-AWARE RECOVERY CHAIN CONTRACT UPDATED

**STATUS:** ACTIVE VARIABLE CONTRACT
**VERSION:** v3.8
**REPLACES:** 03_VARIABLE_CONTRACT_BRIEF_v3.7_LOCKED.txt
**UPDATED FOR:** Business Setup, mode-aware Revenue / COGS, mode-aware Business Summary, Recovery Summary, Cost Allocation, and Business Outcome.

---

## PURPOSE

This brief defines allowed cross-module variable names for QS Tools v3.8.

No module may rename contract variables without updating this brief.

No downstream module may rebuild upstream maths.

No downstream module may consume raw upstream form state unless a locked brief explicitly allows it.

If a downstream module needs a value, the owning upstream module must expose it through its output contract.

---

# 1. CORE CONTRACT RULES

Every cross-module value must have:

```text
one owner
one locked variable name
one clear downstream meaning
one calculation source
one trust state where relevant
```

Benchmarks are diagnostic.

Source module outputs are authoritative.

Predictions are locked separately from actuals.

Cost Summary must consume approved upstream outputs only.

Business Summary must consume Revenue / COGS and Cost Summary outputs only.

Recovery Summary must consume Business Summary outputs only for current reality.

Cost Allocation must consume Recovery Summary outputs for the plan and Labour / Asset identity outputs for structure.

Business Outcome must consume Recovery Summary and Cost Allocation outputs without rebuilding either.

---

# 2. CORE RECOVERY CHAIN CONTRACT PRINCIPLE

The active recovery chain is:

```text
Business Summary = current reality
Recovery Summary = chosen strategy + visible consequence
Cost Allocation = delivery structure + dependency
Business Outcome = verdict
```

Core product rule:

```text
You can choose anything, but you cannot hide from what it causes.
```


Core formulas:

```js

```

```js

```

```js
total_business_cost_annual = total_asset_cost_annual + total_business_overheads
```

```js

```

```js

```

```js
required_recovery_rate = total_productive_output === 0
  ? 0
  : required_revenue / total_productive_output
```

Cost Summary must not calculate required recovery per unit or required recovery per driver.

Business Summary owns the activity-driver bridge.


```js

```

Core recovery split object:

```js
recovery_plan_split = {
  labour_share_percent,
  asset_share_percent,
  overhead_share_percent
}
```

Core component object:

```js
component_required_recovery = {
  labour: {
    share_percent,
    recovery_cost,
    required_recovery_rate
  },
  asset: {
    share_percent,
    recovery_cost,
    required_recovery
  },
  overhead: {
    share_percent,
    recovery_cost
  }
}
```

Core formulas:

```js
recovery_plan_target_per_driver = required_recovery_per_driver
```

```js
labour_recovery_cost = total_cost_burden * labour_share_percent / 100
asset_recovery_cost = total_cost_burden * asset_share_percent / 100
overhead_absorbed_cost = total_cost_burden * overhead_share_percent / 100
```

```js
required_labour_recovery_rate = total_productive_output === 0
  ? 0
  : labour_recovery_cost / total_productive_output
```

```js
required_asset_recovery = asset_recovery_cost
```

Recovery Summary consumes Business Summary outputs.

Recovery Summary must not rebuild Business Summary, Cost Summary, or Revenue / COGS.

Recovery Summary must not calculate allocation_status or business_outcome_status.

# 23. FINAL DEFINITION

The v3.8 variable contract keeps all module boundaries explicit.

Business Setup owns mode.

Revenue / COGS owns Margin Pool and product units.

Cost Summary owns operating cost burden and the legacy hour-based recovery baseline.

Business Summary owns current reality through the selected activity driver.

Recovery Summary owns strategy and visible consequence.

Cost Allocation owns delivery structure and dependency status.

Business Outcome owns the verdict.

No downstream module may rebuild upstream maths or consume unapproved variables.
