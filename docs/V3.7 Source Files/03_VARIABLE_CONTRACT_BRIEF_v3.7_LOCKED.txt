# QS Tools — Variable Contract Brief

## v3.7 — QUOTE ENGINE AND FEEDBACK LOOP CONTRACT UPDATED

**STATUS:** ACTIVE VARIABLE CONTRACT
**VERSION:** v3.7
**REPLACES:** Variable Contract Brief v3.6
**UPDATED FOR:** Quote Engine baseline build, accepted quote snapshots, monthly actuals, and macro feedback loop

---

## PURPOSE

This brief defines allowed cross-module variable names for QS Tools v3.7.

No module may rename contract variables without updating this brief.

No downstream module may rebuild upstream maths.

No downstream module may consume raw upstream form state unless a locked brief explicitly allows it.

If a downstream module needs a value, the owning upstream module must expose it through its output contract.

---

## CORE CONTRACT RULES

Every cross-module value must have:

* one owner
* one locked variable name
* one clear downstream meaning
* one calculation source
* one trust state where relevant

Benchmarks are diagnostic.

Source module outputs are authoritative.

Predictions are locked separately from actuals.

Cost Summary must consume approved upstream outputs only.

Business Summary and Business Modelling must consume locked outputs, not rebuild calculations.

Quote Engine must consume the selected trusted baseline or scenario.

Feedback Loop must compare accepted quote snapshots with monthly actuals without overwriting either.

## COST SUMMARY FORMULAS

```text
total_people_cost_annual =
total_labour_cost_annual


total_business_overheads =
total_general_overheads


total_business_cost_annual =
total_asset_cost_annual + total_business_overheads


total_cost_burden =
total_people_cost_annual
+ total_asset_cost_annual
+ total_business_overheads


required_revenue =
total_cost_burden


required_recovery_rate =
required_revenue / total_productive_output
```

If total_productive_output = 0:

```text
required_recovery_rate = 0
warning required
```

## QUOTE ENGINE FORMULAS

```text
labour_sell_total =
labour_charge_out_rate × base_labour_hours_allowed
```

where entered as rate.

```text
true_labour_cost_total =
base_labour_hours_allowed × model_true_labour_cost_rate
```

```text
direct_cost_total =
material_cost_total
+ true_labour_cost_total
+ direct_cost_package_cost_total
```

```text
quote_margin_pool =
quote_price_total - direct_cost_total
```

```text
business_recovery_required_total =
base_labour_hours_allowed × required_recovery_rate
```

```text
achieved_margin_per_hour =
quote_margin_pool / base_labour_hours_allowed
```

```text
hourly_gap =
achieved_margin_per_hour - required_recovery_rate
```

```text
quote_gap =
quote_margin_pool - business_recovery_required_total
```

```text
max_viable_hours =
quote_margin_pool / required_recovery_rate
```

```text
required_price_at_allowed_hours =
direct_cost_total + business_recovery_required_total
```

```text
price_gap =
required_price_at_allowed_hours - quote_price_total
```

If base_labour_hours_allowed = 0:

* achieved_margin_per_hour = 0
* hourly_gap = 0
* no_labour_hours warning required

If required_recovery_rate = 0:

* max_viable_hours = 0
* no_required_recovery_rate warning required

## FINAL DEFINITION

The v3.7 variable contract keeps all module boundaries explicit.

P&L owns classification, benchmark context, and future monthly actual preparation.

Revenue / COGS owns Margin Pool.

Labour owns people cost and productive output.

General Overheads owns shared / staff / vehicle running / non-asset overheads.

Assets owns ownership-only asset cost and asset finance interest.

Module Reconciliation proves the model.

Model Readiness gates trust.

Cost Summary aggregates approved upstream operating cost outputs.

Business Summary mirrors the current business.

Business Modelling creates locked-baseline scenarios.

Quote Engine tests individual quotes against the selected model.

Accepted Quote Snapshots preserve predictions.

Feedback Loop later compares predictions against monthly actuals.

Cash Flow later owns principal and finance payment pressure.

No downstream module may rebuild upstream maths or consume unapproved variables.
