# QS Tools — Assets Module Brief (v3.6 — LOCKED)

STATUS: ACTIVE SOURCE BRIEF  
VERSION: v3.6

---

## PURPOSE

Assets is the ownership-only asset cost engine.

It calculates the annual cost of owning or financing business assets.

It does not calculate running costs.

---

## SYSTEM POSITION

P&L  
↓  
General Overheads  
↓  
Labour  
↓  
Assets  
↓  
Module Reconciliation  
↓  
Model Readiness  
↓  
Cost Summary

---

## CORE ROLE

Assets owns:

- asset finance cost
- lease cost
- depreciation cost
- ownership cost
- total_asset_cost_annual

---

## INCLUDE

Assets may include:

- finance cost
- lease cost
- depreciation cost
- ownership cost
- annualised replacement / purchase recovery if explicitly treated as ownership cost
- other locked ownership-only asset costs

---

## EXCLUDE

Assets must not include:

- fuel
- servicing
- repairs
- maintenance
- registration
- licensing
- tyres
- consumables
- shared fleet costs
- shared maintenance pools
- vehicle running costs
- direct running cost
- staff assignment
- labour recovery
- pricing
- overhead allocation
- hired equipment / delivery cost

Vehicle running costs belong in General Overheads.

Hired equipment / delivery cost belongs in COGS or delivery cost, not Assets.

---

## OUTPUT CONTRACT

Assets may output:

- total_asset_cost_annual
- assets_ready
- no_active_assets_confirmed

Per asset:

- asset_id
- asset_name
- asset_type
- finance_cost_annual
- lease_cost_annual
- depreciation_cost_annual
- ownership_cost_annual
- total_asset_cost_annual
- is_active

---

## DEPRECATED FIELD

The following field is deprecated in v3.6:

- running_cost_annual

It must not feed Cost Summary.

If still present in code during migration, it must be treated as legacy only and excluded from total_asset_cost_annual unless explicitly migrated into General Overheads.

---

## READINESS RULE

assets_ready = true when one of the following is true:

```text
1. Active assets exist
AND total_asset_cost_annual exists
AND ownership-only rules are followed
AND running_cost_annual does not feed total_asset_cost_annual
AND no vehicle running costs are included
AND no labour costs are included
```

OR:

```text
2. no_active_assets_confirmed = true
AND total_asset_cost_annual = 0
AND no legacy running costs are feeding Assets
```

If no active assets exist and no_active_assets_confirmed is not true, Assets must remain not ready.

---

## NO ACTIVE ASSETS CONFIRMATION RULE

Assets are not automatically ready just because the P&L asset benchmark is missing or zero.

A missing P&L asset benchmark must not be treated as proof that the business has no assets.

Assets may be considered ready only when one of the following is true:

```text
1. Active assets exist and total_asset_cost_annual is calculated using ownership-only costs.

OR

2. The user has explicitly confirmed that no active assets are required for the current business model.
```

The confirmation must be stored as an explicit state/contract value:

```text
no_active_assets_confirmed
```

Do not infer this confirmation from:

```text
missing P&L asset benchmark
zero P&L asset benchmark
zero asset total
empty asset list
incomplete setup data
```

If there are no active assets and no explicit confirmation, Assets must remain not ready.

---

## ASSET READINESS LOGIC

assets_ready = true only when:

```text
active assets exist
AND total_asset_cost_annual exists
AND ownership-only rules are followed
AND asset totals are reconciled or accepted under the readiness rules
```

OR:

```text
no_active_assets_confirmed = true
AND total_asset_cost_annual = 0
AND no legacy running costs are feeding Assets
```

If no_active_assets_confirmed = true, Assets may pass readiness with:

```text
total_asset_cost_annual = 0
```

but Module Reconciliation should still show an informational check stating that the user has confirmed no active assets are required.

---

## IMPORTANT FIRST-PRINCIPLES RULE

Do not assume no assets.

The system must distinguish between:

```text
assets missing because setup is incomplete
```

and:

```text
assets intentionally confirmed as not required
```

Missing data is not confirmation.

Explicit confirmation is required.

---

## COST SUMMARY HANDOFF

Cost Summary must consume:

- total_asset_cost_annual

Cost Summary must not consume:

- running_cost_annual
- fuel
- maintenance
- repair
- vehicle running categories
- active asset-labour links

If no_active_assets_confirmed = true, Cost Summary may consume:

```text
total_asset_cost_annual = 0
```

but this must come from the Assets module contract, not from Cost Summary assuming no assets exist.

---

## NOT ALLOWED

Assets must not:

- assign shared costs
- calculate pricing
- calculate recovery strategy
- calculate productive capacity
- consume Labour totals
- consume General Overheads totals
- expose running cost as a downstream Cost Summary value
- assume no assets are required because asset data is missing
- assume no assets are required because the P&L asset benchmark is zero
- assume no assets are required because the P&L asset benchmark is missing

---

## FINAL DEFINITION

Assets v3.6 is the ownership-only asset cost engine. It produces total_asset_cost_annual and excludes vehicle/fleet running costs, which belong in General Overheads.

Assets may only pass readiness with no active assets when the user explicitly confirms that no active assets are required. Missing data is not confirmation.
