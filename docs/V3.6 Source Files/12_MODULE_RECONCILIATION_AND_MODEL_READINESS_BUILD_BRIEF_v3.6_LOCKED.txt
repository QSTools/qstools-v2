## PURPOSE

Build the system layer that proves whether the upstream QS Tools cost model is complete, internally consistent, and safe to trust before Cost Summary and downstream recovery/pricing pages rely on it.

---

## READINESS FORMULA

model_ready =
pnl_ready
AND general_overheads_ready
AND labour_ready
AND assets_ready
AND reconciliation_ready
AND blocking_checks.length = 0

---

## STATUS FORMULA

If blocking_checks.length > 0:

model_readiness_status = "blocked"

Else if warning_checks.length > 0:

model_readiness_status = "warning"

Else:

model_readiness_status = "ready"

## RECONCILIATION RULES

### P&L Checks

Hard checks:

- P&L data exists
- pnl_ready is true
- unassigned_balance equals 0
- employee_overheads category does not exist
- total_business_costs is available as a benchmark

Warnings:

- total_business_costs is 0
- benchmark totals are unusual or missing but not required for downstream calculation

---

### General Overheads Checks

Hard checks:

- total_general_overheads exists
- general_overheads_ready is true
- category_totals exists
- employee_overheads is not present
- staff_overheads_total is not exposed as a downstream output
- total_general_overheads is the only downstream overhead output

Warnings:

- general_overheads_variance_amount is not 0
- Other / Unallocated category has a balance
- category_totals is empty

Ownership checks:

- vehicle running costs belong in General Overheads
- staff overheads belong in General Overheads as category only
- employer ACC levy does not belong in General Overheads
- asset ownership cost does not belong in General Overheads

---

### Labour Checks

Hard checks:

- total_labour_cost_annual exists
- total_productive_output exists
- total_productive_output is greater than 0
- total_employer_contribution_annual exists
- total_acc_levy_annual exists
- labour_ready is true
- Labour does not expose staff_overheads_total
- Labour does not expose employee_overheads_total

Warnings:

- no active Labour profile exists
- total_labour_cost_annual is 0
- total_acc_levy_annual is 0
- total_productive_output looks unusually low

Ownership checks:

- Labour owns gross wages
- Labour owns entitlements
- Labour owns employer KiwiSaver
- Labour owns ESCT
- Labour owns employer ACC levy
- Labour does not own general overheads
- Labour does not own asset ownership costs
- Labour does not own vehicle running costs

---

### Assets Checks

Hard checks:

- total_asset_cost_annual exists
- assets_ready is true
- running_cost_annual does not feed total_asset_cost_annual
- running_cost_annual is not exposed to Cost Summary
- assets do not contain vehicle running costs

Warnings:

- no active assets exist
- total_asset_cost_annual is 0
- assets benchmark variance exists

Ownership checks:

- Assets owns finance cost
- Assets owns lease cost
- Assets owns depreciation cost
- Assets owns ownership cost
- Assets does not own fuel
- Assets does not own servicing
- Assets does not own repairs
- Assets does not own maintenance
- Assets does not own registration
- Assets does not own tyres
- Assets does not own shared fleet costs
- Assets does not own labour costs

---

### Cross-Module Checks

Hard checks:

- no Employee Overheads module output exists
- no Staff Overheads standalone module output exists
- no banned variable appears in downstream contract
- Cost Summary required inputs are available
- no duplicate cost ownership is detected
- no raw P&L lines are consumed by Cost Summary

Warnings:

- P&L business cost benchmark does not match module total burden
- required_recovery_rate cannot be calculated because productive output is missing or zero
- benchmark variance exists but source-of-truth module totals are present

---

## COST SUMMARY RECONCILIATION RULE

Cost Summary must not own reconciliation.

Cost Summary may display reconciliation status.

Module Reconciliation owns the comparison between:

P&L benchmark totals

and

module source-of-truth totals.

---

## RECONCILIATION TOTALS

Module Reconciliation may calculate diagnostic comparison totals:

module_total_business_costs =
total_labour_cost_annual
+ total_asset_cost_annual
+ total_general_overheads

pnl_business_cost_variance =
module_total_business_costs - total_business_costs

pnl_business_cost_variance_percent =
pnl_business_cost_variance / total_business_costs

These are diagnostics only.

They do not replace module totals.

They do not feed Cost Summary calculations.

---

## COST SUMMARY TRUST RULE

If model_ready = false:

- Cost Summary totals may display
- Cost Summary must show not trusted state
- blocking modules must be visible
- warning modules must be visible
- downstream modules must not treat outputs as final

If model_ready = true and model_readiness_status = "warning":

- Cost Summary may be used
- warnings must remain visible

If model_ready = true and model_readiness_status = "ready":

- Cost Summary may be treated as trusted

## NOT ALLOWED

Module Reconciliation must not:

- edit P&L lines
- move costs automatically
- recalculate Labour totals
- recalculate Assets totals
- recalculate General Overheads totals
- create pricing logic
- create recovery logic

Model Readiness must not:

- create cost totals
- override reconciliation checks
- bypass blocker checks
- hide blocking modules
- allow downstream pages to appear trusted when blocked

## FINAL DEFINITION

Module Reconciliation + Model Readiness Gate v3.6 proves the upstream QS Tools model before Cost Summary is trusted. Reconciliation supplies the evidence. Model Readiness supplies the verdict. Cost Summary consumes the verdict but does not perform the checks itself.
