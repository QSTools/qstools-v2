# QS Tools — First Principles System Brief (v3.6 — LOCKED)

STATUS: ACTIVE SYSTEM PRINCIPLE
VERSION: v3.6

---

## PURPOSE

This brief defines the first-principles operating philosophy behind QS Tools.

It applies to every module, calculation, contract, page, and downstream decision.

QS Tools must not hide assumptions, invent recovery numbers, or create commercial outputs that cannot be traced back to real business costs and real productive capacity.

---

## CORE PRINCIPLE

Simple in hindsight. Invisible beforehand.

QS Tools reflects the business back to the user.

The system exists to make hidden commercial reality visible.

---

## FIRST PRINCIPLES RULE

Every output must be traceable back to:

- a real input
- a stated assumption
- a source module
- a calculation rule
- a downstream contract

If an output cannot be traced, it should not be trusted.

---

## COST OWNERSHIP RULE

Every cost must belong to exactly one module.

If a cost appears in more than one module, the system is incorrect.

If cost ownership is unclear, it must be classified before being used downstream.

---

## ASSUMPTION RULE

The system must remove assumptions wherever feasible.

Where an assumption remains, it must be:

- visible
- named
- explainable
- reviewable
- separated from actuals

Hidden assumptions are not allowed.

---

## NO MAGIC NUMBER RULE

QS Tools must not create a recovery number out of thin air.

A recovery number must come from:

- actual cost burden
- productive capacity
- recovery strategy
- rate model
- real-world quote benchmark

If the number cannot be explained, it is not valid.

---

## MODULE ROLE RULE

Each module must answer one clear question.

P&L:

What does the business financial reality currently show?

General Overheads:

Which shared and unclear costs belong to business overhead?

Labour:

What does labour truly cost and how much productive time exists?

Assets:

What does asset ownership cost?

Module Reconciliation:

Is the upstream model internally consistent?

Model Readiness:

Can the downstream model be trusted?

Cost Summary:

What does the business cost?

Recovery Summary:

How is the cost recovered?

Rate Models:

What rates are required?

Production Capacity:

Can the required output be delivered?

Recovery Outcome:

Is the recovery model commercially viable?

Quote Benchmark:

Does the model align with real quotes?

---

## DOWNSTREAM TRUST RULE

Downstream modules must consume upstream outputs only.

Downstream modules must not rebuild upstream maths.

If an upstream module is incomplete, downstream outputs may display for preview only but must not be treated as trusted.

---

## USER CLARITY RULE

The user must be able to understand:

- where the number came from
- what module owns it
- whether it is actual, assumed, diagnostic, or calculated
- whether it is trusted
- what needs fixing if it is not trusted

---

## RECONCILIATION RULE

The system must distinguish between:

- source-of-truth totals
- benchmark values
- diagnostic comparisons
- warnings
- blockers

A diagnostic comparison must not replace a source-of-truth module output.

---

## COMMERCIAL REALITY RULE

QS Tools must reflect the business as it is, not as the user hopes it is.

If the business cannot recover its cost burden through available productive capacity, the system must show that clearly.

If a quote appears profitable but does not recover the required cost burden, the system must show that clearly.

---

## NOT ALLOWED

QS Tools must not:

- hide assumptions
- duplicate costs
- create untraceable recovery rates
- treat benchmarks as source-of-truth totals
- rebuild upstream maths downstream
- use raw P&L lines directly in Cost Summary
- use display names as join keys
- silently ignore readiness blockers
- present untrusted outputs as final
- reintroduce removed modules without a locked source brief

---

## FINAL DEFINITION

QS Tools v3.6 is a first-principles commercial decision system.

It does not guess what the business should charge.

It shows what the business costs, how those costs are recovered, whether that recovery is operationally possible, and whether real quotes align with the model.

Every number must be traceable, every assumption must be visible, and every cost must belong to exactly one module.
