STATUS: EVIDENCE LAYER  
VERSION: v3.6

---

## PURPOSE

Module Reconciliation checks whether the upstream model is internally consistent, contract-compliant, and safe to pass into Model Readiness.

It is not a user workflow page.

It is a system layer.

---

## CORE PRINCIPLE

Reconciliation is the evidence layer.

Model Readiness is the verdict layer.

Cost Summary must not be trusted until reconciliation has checked the upstream modules.

## HARD CHECKS

### P&L Checks

- P&L data exists
- pnl_ready is true
- required P&L categories are valid
- unassigned_balance = 0
- no active employee_overheads category remains
- P&L can provide benchmark/reference totals if used

---

### General Overheads Checks

- total_general_overheads exists
- general_overheads_ready is true
- General Overheads categories exist
- Staff Overheads is category only
- Staff Overheads is not a separate module
- total_general_overheads is the only downstream overhead output
- vehicle running costs are classified to General Overheads
- General Overheads does not include Labour employer obligations
- General Overheads does not include asset ownership costs

---

### Labour Checks

- total_labour_cost_annual exists
- total_productive_output exists
- total_productive_output > 0
- total_employer_contribution_annual exists
- employer KiwiSaver is included
- ESCT is included
- employer ACC levy is included
- Labour does not include staff overheads
- Labour does not include asset ownership costs
- Labour does not include general overheads

---

### Assets Checks

- total_asset_cost_annual exists
- assets_ready is true
- Assets are ownership-only
- running_cost_annual does not feed Cost Summary
- Assets do not include fuel
- Assets do not include servicing
- Assets do not include maintenance
- Assets do not include registration
- Assets do not include tyres
- Assets do not include shared fleet costs
- Assets do not include labour cost

If no active assets exist, Module Reconciliation must check whether the user has explicitly confirmed that no active assets are required.

Allowed confirmation field:

```text
no_active_assets_confirmed
```

Rules:

```text
If active assets exist:
  Assets readiness should be based on ownership-only cost completeness and reconciliation.

If no active assets exist and no_active_assets_confirmed = true:
  Assets may pass readiness with an informational check.

If no active assets exist and no confirmation exists:
  Assets readiness must fail.
```

A missing or zero P&L asset benchmark must not be treated as confirmation that no assets are required.

---

### No Active Assets Confirmation Check

If no active assets exist, Module Reconciliation must distinguish between incomplete setup and intentional no-asset setup.

```text
No active assets + no confirmation = blocker
No active assets + explicit confirmation = pass with info
Active assets + valid ownership-only total = pass or warning based on reconciliation
```

This rule exists to avoid hidden assumptions.

---

### Cross-Module Checks

- no Employee Overheads output exists
- no Staff Overheads standalone output exists
- no duplicate cost ownership detected
- Cost Summary inputs are available
- banned variables are not present in downstream contracts
- no raw P&L lines are consumed by Cost Summary

## BLOCKER VS WARNING RULE

### Blockers

A blocker means Cost Summary may display totals but must not be trusted.

Examples:

- missing total_labour_cost_annual
- missing total_general_overheads
- missing total_asset_cost_annual
- employer ACC levy missing from Labour
- running_cost_annual feeding Assets total
- employee_overheads active category or output detected
- total_productive_output missing or zero where recovery rate is required
- no active assets exist and no_active_assets_confirmed is not true
- module total business cost differs materially from P&L business cost

---

### Warnings

A warning means the model can display but the user should review the result.

Examples:

- P&L benchmark variance exists
- unallocated General Overheads remain
- optional category detail missing
- unusual cost split detected
- no active assets exist but no_active_assets_confirmed is true
- P&L asset benchmark is missing but the user has explicitly confirmed no active assets are required

## DIAGNOSTIC TOTALS

Module Reconciliation may calculate:

```text
module_total_business_costs =
total_labour_cost_annual
+ total_asset_cost_annual
+ total_general_overheads
```

```text
pnl_business_cost_variance =
module_total_business_costs - total_business_costs
```

```text
pnl_business_cost_variance_percent =
pnl_business_cost_variance / total_business_costs
```

These are diagnostics only.

They do not replace module totals.

They do not feed Cost Summary calculations.

---

## NOT ALLOWED

Module Reconciliation must not:

- recalculate Labour
- recalculate Assets
- recalculate General Overheads
- edit P&L lines
- move costs automatically
- create pricing logic
- create recovery logic
- create new source-of-truth totals
- infer that no assets are required from missing asset data
- infer that no assets are required from a missing or zero P&L asset benchmark

---

## FINAL DEFINITION

Module Reconciliation v3.6 is the evidence layer that checks whether P&L, General Overheads, Labour, and Assets are contract-compliant before Model Readiness issues the final trust verdict.

Assets may only pass without active asset records when the user explicitly confirms that no active assets are required. Missing asset data is not confirmation.
