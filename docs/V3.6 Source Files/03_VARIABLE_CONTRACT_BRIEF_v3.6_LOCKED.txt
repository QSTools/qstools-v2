# QS Tools — Variable Contract Brief (v3.6 — LOCKED)

STATUS: ACTIVE VARIABLE CONTRACT
VERSION: v3.6
UPDATED FOR: FORWARD-LOOKING COST SUMMARY + REVENUE / COGS FLOW + QUOTE ENGINE BOUNDARY

---

## PURPOSE

This brief defines allowed cross-module variable names for QS Tools v3.6.

No module may rename contract variables without updating this brief.

No downstream module may rebuild upstream maths.

No downstream module may consume raw upstream form state unless a locked brief explicitly allows it.

If a downstream module needs a value, the owning upstream module must expose it through its output contract.

---

## CORE CONTRACT RULES

Every cross-module value must have:

- one owner
- one locked variable name
- one clear downstream meaning
- one calculation source

Benchmarks are diagnostic.

Source module outputs are authoritative.

Cost Summary must consume approved upstream outputs only.

Business Summary and Business Modelling must consume locked outputs, not rebuild calculations.

Quote Engine must consume the selected trusted baseline or scenario when eventually built.

## COST SUMMARY FORMULAS

total_people_cost_annual =
total_labour_cost_annual

total_business_overheads =
total_general_overheads

total_business_cost_annual =
total_asset_cost_annual + total_business_overheads

total_cost_burden =
total_people_cost_annual
+ total_asset_cost_annual
+ total_business_overheads

required_revenue =
total_cost_burden

required_recovery_rate =
required_revenue / total_productive_output

If total_productive_output = 0:

required_recovery_rate = 0
warning required.

## REVENUE / COGS OUTPUTS

Allowed Revenue / COGS outputs:

- total_revenue
- total_direct_costs
- margin_pool
- gross_margin_percent
- revenue_cogs_ready
- revenue_cogs_warnings

Optional display outputs:

- revenue_category_totals
- direct_cost_category_totals

Core formula:

margin_pool =
total_revenue - total_direct_costs

gross_margin_percent =
margin_pool / total_revenue

If total_revenue = 0:

gross_margin_percent = 0
warning required.

Revenue / COGS consumes classified P&L income and COGS / direct-cost outputs only.

Revenue / COGS must not consume Labour, Assets, General Overheads, Cost Summary internals, or scenario outputs.

## FINAL DEFINITION

The v3.6 variable contract keeps all module boundaries explicit.

P&L owns classification and benchmark context.

Labour owns people cost and productive output.

General Overheads owns shared / staff / vehicle running / non-asset overheads.

Assets owns ownership-only asset cost and asset finance interest.

Module Reconciliation proves the model.

Model Readiness gates trust.

Cost Summary aggregates approved upstream operating cost outputs.

Revenue / COGS produces the margin pool.

Business Summary mirrors the current business.

Business Modelling creates locked-baseline scenarios.

Quote Engine is a future quote-level commercial test layer governed by 22_QUOTE_ENGINE_v3.6_LOCKED.txt.

Cash Flow later owns principal and finance payment pressure.

No downstream module may rebuild upstream maths or consume unapproved variables.
