# Mirra — Job Costing / Profitability Brief

## v4.0 — ACTUAL JOB PROFITABILITY, QUOTED VS ACTUAL, MODEL VARIANCE, RATE FEEDBACK, AND BUSINESS OUTCOME FEEDBACK

STATUS: FUTURE BUILD BRIEF
VERSION: v4.0
FILE NUMBER: 23

REPLACES:
- older assumptions that job profitability can be proven from quote totals alone
- older assumptions that quoted margin equals actual margin
- older logic that does not compare actual job consumption against model targets
- older logic that does not feed actual job outcomes back to Rate Builder and Business Outcome
- older logic that treats PO / invoice / labour / asset usage as disconnected from quote validation

PRODUCT NAME:
Mirra

FORMER NAME:
QS Tools

PRIMARY RULE:
Job Costing / Profitability compares quoted, modelled, ordered, invoiced, consumed, and actual job results. It proves whether the business model and quote assumptions worked in real jobs. It does not rewrite source modules automatically.

---

# 1. PURPOSE

Job Costing / Profitability answers:

Did the job make money?
Did the job recover the required cost?
Did the quote align with the approved model?
Did actual cost exceed the quote?
Did labour take longer than expected?
Did assets recover what they needed to recover?
Did materials cost more than allowed?
Did subcontractors exceed allowance?
Did the job hit the target margin?
Which quote lines failed?
Which model assumptions were wrong?
What should feed back into Rate Builder and Business Outcome?

This layer is where Mirra proves the model against real completed work.

# 3. CORE PRINCIPLE

Quote Checker proves whether the quote was aligned before work starts.

Job Costing proves whether the job actually worked after work happens.

These are different checks.

Quote Checker asks:

Was the quote safe?

Job Costing asks:

Was the job profitable?

A quote can pass and a job can still fail if actual delivery is worse than expected.

A quote can fail and a job can still make money, but that should be treated as risk or luck, not trusted model alignment.

# 19. PROFITABILITY FORMULAS

quoted_revenue =
sum quoted_total

total_actual_cost =
actual_material_cost
+ actual_labour_cost
+ actual_asset_cost
+ actual_subcontractor_cost
+ actual_overhead_recovery

actual_gross_margin =
quoted_revenue - actual_material_cost - actual_subcontractor_cost

actual_net_margin =
quoted_revenue - total_actual_cost

actual_gross_margin_percent =
actual_gross_margin / quoted_revenue

actual_net_margin_percent =
actual_net_margin / quoted_revenue

Do not divide by zero.

If quoted_revenue = 0:

margin percentages = 0
status = review_required or blocked

---

# 20. MODEL VARIANCE FORMULAS

variance_to_quote =
quoted_revenue - total_actual_cost

variance_to_model_minimum =
quoted_revenue - model_minimum_required_recovery

variance_to_model_target =
quoted_revenue - model_target_revenue

If variance_to_model_minimum < 0:

job under-recovered minimum required cost.

If variance_to_model_target < 0:

job may recover cost but missed target.

# 21. ACTUAL VS MODEL STATUS

Allowed actual_vs_model_status values:

- passed_model
- passed_minimum_only
- under_recovered
- margin_shortfall
- review_required
- blocked
- not_available

Definitions:

passed_model:
Actual job result meets or exceeds selected model target.

passed_minimum_only:
Actual job result meets minimum recovery but misses target margin.

under_recovered:
Actual job result fails minimum recovery.

margin_shortfall:
Actual job recovers cost but misses target margin.

review_required:
Data exists but mapping or quantity review is required.

blocked:
Required source data is invalid.

not_available:
Actual data not yet available.

---

# 22. JOB PROFITABILITY STATUS VALUES

Allowed job_profitability_status values:

- profitable
- profitable_below_target
- break_even
- under_recovered
- loss
- review_required
- blocked
- preview_only
- not_ready

Definitions:

profitable:
Actual job net margin is positive and meets target.

profitable_below_target:
Actual job is profitable but below target.

break_even:
Actual job approximately equals actual cost.

under_recovered:
Actual job fails minimum recovery.

loss:
Actual job net margin is negative.

review_required:
Actuals need review.

blocked:
Required actual data is invalid.

preview_only:
Values display but source trust is not final.

not_ready:
Job actuals are missing.

---

# 23. BAD RESULT IS NOT DATA FAILURE

A job showing a loss is not automatically bad data.

A job showing under-recovery is not automatically bad data.

These are commercial results if source data is trusted.

Only missing, invalid, unmapped, or inconsistent data should block.

Allowed values:

- ready
- warning
- blocked
- preview_only

Mapping:

If SOQ / Job / PO source is blocked:
job_costing_model_trust_state = blocked

If Quote Checker targets are blocked:
job_costing_model_trust_state = blocked

If actual data is incomplete:
job_costing_model_trust_state = warning or preview_only

If job result is loss but data is trusted:
job_costing_model_trust_state = warning

If all actuals are mapped and valid:
job_costing_model_trust_state = ready
