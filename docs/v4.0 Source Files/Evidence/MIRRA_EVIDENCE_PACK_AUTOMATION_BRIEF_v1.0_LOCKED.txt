# Mirra — Evidence Pack Automation Brief

## v1.0 — PYTHON SCRIPT FOR CHANGELOGS, SCREENSHOTS, README, INDEXING, AND MILESTONE EVIDENCE PACKS

STATUS: BUILD BRIEF
VERSION: v1.0
FILE NUMBER: EVIDENCE-01

PRODUCT NAME:
Mirra

FORMER NAME:
QS Tools

PRIMARY RULE:
The Evidence Pack Automation script creates a dated milestone evidence folder that collects changelog files, screenshots, README notes, and index metadata into one clean package. It reduces manual documentation workload while preserving product development evidence.

---

# 1. PURPOSE

This script exists to make milestone documentation fast.

It should allow the user to:

1. Run changelog commands.
2. Take screenshots.
3. Drop screenshots into a temporary folder.
4. Run one Python command.
5. Automatically create a complete evidence pack.

The script should prevent manual matching of screenshots to logs.

---

# 2. WHY THIS EXISTS

Mirra is becoming a real product.

The system now needs lightweight evidence of:

- major architecture decisions
- brief pack updates
- module rebuilds
- before / after UI states
- code changes
- source pack changes
- product concept milestones

The evidence process must not slow down development.

Python should do the filing, naming, copying, and indexing.

---

# 3. TARGET WORKFLOW

The normal milestone workflow is:

```txt
1. Run changelog command
2. Take screenshots
3. Save screenshots into EvidenceDrop folder
4. Run Python evidence pack script
5. Review generated evidence pack
6. Commit or archive if milestone is important
