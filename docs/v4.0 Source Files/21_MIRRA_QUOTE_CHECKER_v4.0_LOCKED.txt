## v4.0 — QUOTE ALIGNMENT, MODEL TARGET ENFORCEMENT, MINIMUM RECOVERABLE RATE CHECKING, MARGIN VALIDATION, AND NO-SOURCE-MUTATION BOUNDARY

PRODUCT NAME:
Mirra

FORMER NAME:
QS Tools

PRIMARY RULE:
Quote Checker checks whether a quote aligns with the approved business model, minimum recoverable rates, selected Business Outcome targets, and Rate Builder assumptions. It does not create source costs, rebuild rates, change model assumptions, or issue source module corrections.

---

# 1. PURPOSE

Quote Checker answers:

Does this quote align with the business model?
Does this quote recover the minimum required cost?
Does this quote use the correct rate?
Does this quote meet the selected margin / markup target?
Does this quote align with Business Outcome modelling assumptions?
Does this quote under-recover labour, assets, overheads, or materials?
Which quote lines are below minimum recoverable rate?
Which quote lines are missing required cost coverage?
Does the full quote pass or fail?

Quote Checker is the enforcement layer.

Business Outcome sets the target.

Quote Checker tests the quote against that target.

# 3. CORE PRINCIPLE

Quote Checker does not decide what the business should be.

It checks whether a quote matches what the model says the business needs.

It must compare actual quoted rates and quote line values against:

- minimum recoverable rates
- suggested sell rates
- selected Business Outcome targets
- required margin
- required markup
- component cost assumptions
- material markup assumptions
- labour recovery assumptions
- asset recovery assumptions
- overhead recovery assumptions

Quote Checker should make quote failure obvious.

# 13. RATE MATCHING RULE

Each quote line should be matched to a Rate Builder rate where possible.

Matching may use:

- explicit rate_id
- rate_name
- rate_type
- unit
- description mapping
- user-selected match
- future stored quote template mapping

If no match exists:

line_status = review_required

Do not guess silently.

Do not force a match.

---

# 14. QUOTE LINE CHECK FORMULA

For each quote line:

quoted_unit_rate =
quoted_total / quantity

If quantity = 0:

quoted_unit_rate = 0
line_status = blocked or review_required

Minimum recovery check:

line_recovery_gap =
quoted_unit_rate - model_minimum_recoverable_rate

If line_recovery_gap < 0:

line is under-recovering.

Suggested sell rate check:

line_sell_rate_gap =
quoted_unit_rate - model_suggested_sell_rate

If line_sell_rate_gap < 0:

line may recover cost but miss target margin.

---

# 15. LINE PASS STATUS VALUES

Allowed quote_line_pass_status values:

- pass
- pass_with_warning
- under_recovering
- margin_shortfall
- unmatched
- review_required
- blocked
- not_available

Definitions:

pass:
Line meets or exceeds minimum recoverable and target sell rate.

pass_with_warning:
Line meets minimum recovery but has non-blocking warnings.

under_recovering:
Line is below minimum recoverable rate.

margin_shortfall:
Line is above minimum recoverable rate but below suggested sell rate or margin target.

unmatched:
No Rate Builder match exists.

review_required:
Line can be checked only partially.

blocked:
Calculation cannot run due to missing quantity, invalid input, or blocked upstream target.

not_available:
Required model target not available.

---

# 16. QUOTE TOTAL CHECKS

Quote Checker must calculate:

quote_total_revenue =
sum quote_lines.quoted_total

quote_minimum_required_recovery =
sum line minimum recoverable amount

quote_required_target_revenue =
sum line suggested target amount

quote_recovery_gap =
quote_total_revenue - quote_minimum_required_recovery

quote_target_gap =
quote_total_revenue - quote_required_target_revenue

quote_margin_amount =
quote_total_revenue - quote_minimum_required_recovery

quote_margin_percent =
quote_margin_amount / quote_total_revenue

Do not divide by zero.

---

# 17. QUOTE PASS STATUS VALUES

Allowed quote_checker_status values:

- pass
- pass_with_warnings
- under_recovering
- margin_shortfall
- review_required
- blocked
- preview_only
- not_ready

Definitions:

pass:
Quote meets minimum recovery and selected target assumptions.

pass_with_warnings:
Quote passes but warnings remain.

under_recovering:
Quote total or one or more lines are below minimum recoverable rate.

margin_shortfall:
Quote recovers cost but misses margin / markup target.

review_required:
Quote can be checked but needs user review.

blocked:
Required source target or quote input is invalid.

preview_only:
Quote check displays but source trust is not final.

not_ready:
Required quote data is missing.

# 18. MINIMUM RECOVERY RULE

Minimum recovery is non-negotiable.

If quoted rate is below minimum recoverable rate:

- flag under_recovering
- show shortfall
- show affected line
- show model minimum
- show quoted rate
- show amount required to pass

This is not a data failure.

It is a commercial warning.

---

# 19. TARGET SELL RATE RULE

Suggested sell rate includes margin/profit target.

If quoted rate is above minimum recoverable but below suggested sell rate:

- flag margin_shortfall
- show target gap
- show required margin
- show actual margin
- show amount needed to meet target

This is less severe than under-recovery but still important.

---

# 20. MATERIAL MARKUP CHECK

Where quote lines include material or product/unit items and a material markup target exists:

Quote Checker should compare:

- quoted material markup
- target material markup
- blended material markup assumption
- modelled supplier / COG scenario assumptions

Important:

If material line detail is not available, show not_available.

Do not infer material line cost from high-level supplier totals.

Do not pretend supplier-specific markup exists unless the quote line has source cost detail.

---

# 21. LABOUR RATE CHECK

For labour lines:

Compare quoted labour rate to:

- labour minimum recoverable rate
- labour suggested sell rate
- selected labour rate from Rate Builder

Flag:

- under-recovering labour
- margin shortfall labour
- unmatched labour role
- quantity/hours missing

---

# 22. ASSET / MACHINE RATE CHECK

For asset or machine + operator lines:

Compare quoted rate to:

- asset component recovery
- operator component recovery where included
- overhead burden
- suggested machine/operator rate from Rate Builder

Example:

Digger + Operator quote line must be checked against the approved Digger + Operator rate.

Do not check machine-only if the approved rate includes operator unless line is explicitly machine-only.

---

# 23. FULL CREW RATE CHECK

For full crew quote lines:

Compare against matching full group or crew rate from Rate Builder.

Full crew may include:

- multiple labour roles
- one or more assets
- overhead burden
- margin

If only some crew components are quoted separately, check line by line and total quote level.

---

# 24. PRODUCT / UNIT RATE CHECK

For product/unit lines:

If product-based business or valid SOQ/product structure exists:

Compare against:

- product/unit minimum recoverable rate
- product/unit suggested sell rate
- material/COG assumptions
- labour/unit assumptions
- asset/unit assumptions
- overhead/unit assumptions

If labour-based business and no SOQ/product structure exists:

Do not attempt detailed unit validation.

Show review_required or not_available.

---

# 25. MODE-AWARE QUOTE CHECKING

Quote Checker must be mode-aware.

Labour-based business default checks:

- hourly labour rates
- machine/operator rates
- crew rates
- daywork rates
- product/unit rates only if manually or SOQ validated

Product-based business default checks:

- unit/product rates
- material markup assumptions
- unit margin
- daywork/hourly exception rates where present

Mode guides the default checking behaviour.

It does not prevent exception rates.

---

# 30. TRUST STATE

Quote Checker must expose:

quote_checker_model_trust_state

Allowed values:

- ready
- warning
- blocked
- preview_only

Mapping:

If Business Outcome targets are blocked:
quote_checker_model_trust_state = blocked

If Rate Builder rows are blocked:
quote_checker_model_trust_state = blocked

If quote lines are invalid:
quote_checker_model_trust_state = blocked or warning depending severity

If quote under-recovers but source data is trusted:
quote_checker_model_trust_state = warning

If source data is preview-only:
quote_checker_model_trust_state = preview_only

If all checks pass:
quote_checker_model_trust_state = ready
