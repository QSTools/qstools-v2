# Mirra — Business Modelling Brief

## v4.0 — TARGET MODEL CREATION, SCENARIO LEVERS, SUPPLIER / MATERIAL SENSITIVITY, RATE TARGET SELECTION, AND QUOTE CHECKER BRIDGE

STATUS: ACTIVE / BRIDGE SOURCE BRIEF
VERSION: v4.0
FILE NUMBER: 30

REPLACES:
- older assumptions that Business Outcome directly feeds Quote Checker
- older assumptions that Business Outcome is the modelling page
- older assumptions that Quote Checker consumes unapproved or unstructured scenario values
- older logic where modelling changes are mixed into Rate Builder
- older logic where supplier/material sensitivity has no target handoff
- older logic where quote checking has no selected business model target

PRODUCT NAME:
Mirra

FORMER NAME:
QS Tools

PRIMARY RULE:
Business Modelling is the bridge between Business Outcome and Quote Checker. It turns diagnosed business problems into selected target assumptions, modelled rates, recovery targets, and quote-checking rules. It does not create source truth, rebuild source modules, or check quotes.

---

# 1. PURPOSE

Business Modelling answers:

What are we going to change?
What target model should the business use?
Which rate changes are required?
Which supplier/category assumptions are being modelled?
Which labour productivity assumptions are being modelled?
Which asset utilisation assumptions are being modelled?
Which overhead assumptions are being modelled?
What is the expected result if those changes are applied?
What targets should Quote Checker enforce?

Business Outcome says:

This is the problem.

Business Modelling says:

This is the selected fix.

Quote Checker says:

Does this quote follow the selected fix?

Rate Builder creates base rates.

Business Modelling may adjust or target those rates.

Example:

Business Modelling may test:

- increase current rate to $125/hr
- increase all labour rates by 5%

- accept Rate Builder suggested rates
- set a custom target rate

Business Modelling must not rebuild the rate component formula.

It uses Rate Builder outputs as the base.

---

# 16. SUPPLIER MODELLING FORMULA

supplier_adjustment_amount =
supplier_spend × adjustment_percent

modelled_supplier_spend =
supplier_spend + supplier_adjustment_amount

modelled_total_cog =
current_total_cog + supplier_adjustment_amount

modelled_margin_pool =
total_revenue - modelled_total_cog

modelled_gross_margin_percent =
modelled_margin_pool / total_revenue

If total_revenue = 0:

modelled_gross_margin_percent = 0
status = review_required or blocked

Negative adjustment percent means supplier cost reduction.

Positive adjustment percent means supplier cost increase.

---

# 17. CATEGORY MODELLING FORMULA

category_adjustment_amount =
category_spend × adjustment_percent

modelled_total_cog =
current_total_cog + category_adjustment_amount

modelled_margin_pool =
total_revenue - modelled_total_cog

modelled_gross_margin_percent =
modelled_margin_pool / total_revenue

This shows the effect of category-level movement.

# 28. MODEL STATUS VALUES

Allowed business_modelling_status values:

- ready
- ready_with_warnings
- review_required
- blocked
- preview_only
- target_selected
- not_ready

Definitions:

ready:
Scenarios can be modelled.

target_selected:
A target model has been selected for Quote Checker.

ready_with_warnings:
Modelling works but warnings remain.

review_required:
Scenario values need review.

blocked:
Required upstream values are blocked.

preview_only:
Scenario displays but is not trusted.

not_ready:
Required upstream outputs are missing.

---

# 29. TRUST STATE

Business Modelling must expose:

business_modelling_model_trust_state

Allowed values:

- ready
- warning
- blocked
- preview_only

Mapping:

If Business Outcome is blocked:
trust_state = blocked

If Rate Builder is blocked:
trust_state = blocked

If key upstream source is preview-only:
trust_state = preview_only

If scenario values require review:
trust_state = warning

If target model selected with trusted source:
trust_state = ready

# 31. WARNING MEANINGS

## business_modelling_supplier_markup_not_individual

Use when supplier rows are shown but user may interpret them as supplier-specific markups.

Blocking:

false.

Meaning:

Supplier/category rows show contribution to blended material markup, not true supplier-level markup.

## business_modelling_product_unit_requires_soq

Use when a labour-based business tries to model product/unit costs without manual components or SOQ/job data.

Blocking:

false or true depending attempted output.

Meaning:

True product/unit modelling needs physical quantity data.

## business_modelling_rate_target_below_minimum

Use when a selected target rate is below minimum recoverable rate.

Blocking:

false by default, but target must be clearly marked under-recovering.

## business_modelling_target_not_selected

Use when modelling scenarios exist but no target model has been selected for Quote Checker.

Blocking:

true for Quote Checker handoff.
